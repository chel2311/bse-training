# 演習2: ブランチとマージ

## 状況設定
オフショア開発チームから「計算機アプリ」のベースコードが納品されました。
あなたは新機能（剰余計算）をfeatureブランチで追加し、mainブランチにマージする練習をします。

## 前提条件
- 演習1が完了していること（git clone〜pushの基本操作を理解していること）
- 練習用のローカルリポジトリがあること

---

## 手順

### Step 1: 準備

演習1で作成したリポジトリ、または新しいリポジトリで作業する。

```bash
# リポジトリのフォルダに移動
cd C:\work\git-training

# 最新の状態を取得
git pull

# Calculator.cs をリポジトリに追加
# （練習用ファイル/Calculator.cs をコピー）
git add Calculator.cs
git commit -m "計算機クラスのベースコードを追加"
git push
```

### Step 2: 現在のブランチを確認

```bash
# ブランチ一覧を表示
git branch
```

出力例:
```
* main
```
→ `*` がついているのが現在のブランチ。mainブランチにいることを確認。

### Step 3: featureブランチを作成

```bash
# 新しいブランチを作成して切り替え
git checkout -b feature/add-modulo
```

出力例:
```
Switched to a new branch 'feature/add-modulo'
```

```bash
# ブランチ一覧で確認
git branch
```

出力例:
```
* feature/add-modulo
  main
```
→ `feature/add-modulo` に切り替わった。

### Step 4: featureブランチで機能を追加

Calculator.cs を開き、剰余計算メソッドを追加する。
（`練習用ファイル/Calculator_feature.cs` を参考にする）

**追加する内容**:

```csharp
/// <summary>
/// 剰余（余り）を計算する
/// </summary>
public int Modulo(int a, int b)
{
    if (b == 0)
        throw new DivideByZeroException("0で割ることはできません");
    return a % b;
}
```

### Step 5: featureブランチでコミット

```bash
# 変更を確認
git status

# 差分を確認
git diff

# ステージング
git add Calculator.cs

# コミット
git commit -m "剰余計算（Modulo）メソッドを追加"
```

### Step 6: mainブランチに切り替え

```bash
# mainブランチに切り替え
git checkout main
```

ここで Calculator.cs を確認すると、featureブランチで追加した `Modulo` メソッドがないことがわかる。
→ ブランチごとに異なるコードの状態を持てることを体験する。

```bash
# mainブランチに戻ったことを確認
git branch
```

出力例:
```
  feature/add-modulo
* main
```

### Step 7: featureブランチをマージ

```bash
# feature/add-modulo の変更を main に取り込む
git merge feature/add-modulo
```

出力例:
```
Updating abc1234..def5678
Fast-forward
 Calculator.cs | 10 ++++++++++
 1 file changed, 10 insertions(+)
```

ここで Calculator.cs を確認すると、`Modulo` メソッドが追加されていることがわかる。

### Step 8: マージ結果を確認

```bash
# 履歴を確認
git log --oneline

# リモートに送信
git push
```

### Step 9: 不要になったブランチを削除（任意）

```bash
# マージ済みのブランチを削除
git branch -d feature/add-modulo
```

## ブランチの流れ（図解）

```
main:              o───────────────o───── (マージ後)
                   │               ↑
feature/add-modulo:│   o───o───o───┘
                   │   ↑
                   │   │
                  Step3で     Step4-5で    Step7で
                  ブランチ作成  機能追加     マージ
```

## よくある質問

**Q: ブランチ名はどう付ければよいか？**
A: プロジェクトのルールに従う。一般的には `feature/機能名`、`bugfix/バグ内容` の形式。

**Q: mainブランチで直接作業してはいけないのか？**
A: チーム開発ではmainブランチへの直接コミットは避ける。
必ずfeatureブランチを作り、Pull Requestを経由してマージする。

**Q: マージ後にfeatureブランチは消すべきか？**
A: マージが完了し不要になったブランチは削除して整理するのが一般的。
