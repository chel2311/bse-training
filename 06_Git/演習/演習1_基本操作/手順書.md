# 演習1: Git基本操作

## 状況設定
あなたはBSEとして、オフショア開発チームが使用しているGitリポジトリに参加することになりました。
まずはGitの基本操作（clone → 編集 → add → commit → push）の一連の流れを練習します。

## 前提条件
- GitがPCにインストールされていること
- GitHubまたはAzure DevOpsのアカウントがあること
- Visual Studioがインストールされていること

---

## 手順

### Step 1: リポジトリの準備

**GitHubで練習用リポジトリを作成する場合:**

1. GitHub（https://github.com）にログイン
2. 右上の「+」→「New repository」
3. Repository name: `git-training`
4. 「Add a README file」にチェック
5. 「Create repository」をクリック

### Step 2: リポジトリをクローン

コマンドプロンプトまたはGit Bashで以下を実行:

```bash
# 作業用フォルダに移動
cd C:\work

# リポジトリをクローン（URLは自分のリポジトリのURL）
git clone https://github.com/あなたのユーザー名/git-training.git

# クローンしたフォルダに移動
cd git-training
```

**Visual Studioで行う場合:**
1. ファイル → リポジトリのクローン
2. リポジトリURLを入力
3. ローカルパスを指定（例: `C:\work\git-training`）
4. 「クローン」をクリック

### Step 3: 現在の状態を確認

```bash
# 状態を確認
git status
```

出力例:
```
On branch main
nothing to commit, working tree clean
```
→ まだ何も変更していないので「clean（きれい）」と表示される。

### Step 4: ファイルを追加

`練習用ファイル/Sample.cs` をリポジトリのフォルダにコピーする。

```bash
# 状態を確認
git status
```

出力例:
```
On branch main
Untracked files:
  (use "git add <file>..." to include in what will be committed)
        Sample.cs

nothing added to commit but untracked files present
```
→ `Sample.cs` が「Untracked（未追跡）」として検出される。

### Step 5: ファイルをステージング

```bash
# Sample.cs をステージングに追加
git add Sample.cs

# 状態を確認
git status
```

出力例:
```
On branch main
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        new file:   Sample.cs
```
→ `Sample.cs` がコミット対象になった。

### Step 6: コミット

```bash
# コミットメッセージ付きでコミット
git commit -m "顧客クラスのサンプルファイルを追加"
```

出力例:
```
[main abc1234] 顧客クラスのサンプルファイルを追加
 1 file changed, 35 insertions(+)
 create mode 100644 Sample.cs
```

### Step 7: ファイルを編集

Sample.cs をテキストエディタまたはVisual Studioで開き、以下の変更を加える:

**変更内容**: `GetDisplayInfo` メソッドに電話番号を追加表示する

```csharp
// 変更前
public string GetDisplayInfo()
{
    return $"[{Id}] {Name}（{Age}歳）";
}

// 変更後
public string GetDisplayInfo()
{
    return $"[{Id}] {Name}（{Age}歳）TEL: {Phone}";
}
```

### Step 8: 差分を確認

```bash
# 変更内容を確認
git diff
```

出力例:
```diff
diff --git a/Sample.cs b/Sample.cs
--- a/Sample.cs
+++ b/Sample.cs
@@ -25,7 +25,7 @@ public class Customer
     public string GetDisplayInfo()
     {
-        return $"[{Id}] {Name}（{Age}歳）";
+        return $"[{Id}] {Name}（{Age}歳）TEL: {Phone}";
     }
```

### Step 9: 変更をコミット

```bash
# 変更をステージング
git add Sample.cs

# コミット
git commit -m "GetDisplayInfoメソッドに電話番号の表示を追加"
```

### Step 10: リモートに送信

```bash
# リモートリポジトリにプッシュ
git push
```

### Step 11: 履歴を確認

```bash
# コミット履歴を確認
git log --oneline
```

出力例:
```
def5678 GetDisplayInfoメソッドに電話番号の表示を追加
abc1234 顧客クラスのサンプルファイルを追加
xyz9876 Initial commit
```

## 確認ポイント

以下の内容が理解できていれば、この演習はクリアです:

- [ ] `git clone` でリポジトリを取得できた
- [ ] `git status` で現在の状態を確認する習慣がついた
- [ ] `git add` → `git commit` → `git push` の流れが理解できた
- [ ] `git diff` で変更内容を確認できた
- [ ] `git log` でコミット履歴を確認できた
- [ ] コミットメッセージに「何を変更したか」を書く重要性が理解できた

## Visual Studioでの同等操作

コマンドラインが苦手な場合、Visual Studioで同じ操作が可能です:

| コマンド | Visual Studio操作 |
|---------|------------------|
| git clone | ファイル → リポジトリのクローン |
| git status | Git Changes ウィンドウで確認 |
| git add | 変更ファイルの「+」ボタン |
| git commit | メッセージ入力 →「コミット」ボタン |
| git push | 「プッシュ」ボタン（↑アイコン） |
| git diff | 変更ファイルをダブルクリック |
| git log | Git → コミット履歴の表示 |
