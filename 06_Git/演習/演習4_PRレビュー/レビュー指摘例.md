### 問題1: ファイルパスのハードコーディング

**該当コード**: `FormMain.cs` btnExportCsv_Click メソッド内

```csharp
// 現在の実装（問題あり）
string filePath = @"C:\export\customers.csv";
```

**何が問題か**:
- 出力先パスが `C:\export\` に固定されている
- このフォルダが存在しない環境ではエラーになる（`DirectoryNotFoundException`）
- ユーザーが出力先を選択できない
- 他のPCやサーバーに配備した際に動作しない

**修正案**:
```csharp
// SaveFileDialogで出力先をユーザーに選択させる
using (var dialog = new SaveFileDialog())
{
    dialog.Filter = "CSVファイル (*.csv)|*.csv";
    dialog.FileName = "customers.csv";
    dialog.Title = "CSVエクスポート先を選択";

    if (dialog.ShowDialog() != DialogResult.OK)
        return;

    string filePath = dialog.FileName;
    // 以降の処理...
}
```

**レビュー指摘例**:
```

【分類】改善提案 / セキュリティ
```

### 問題2: ファイルストリームがusingで囲まれていない（リソースリーク）

**該当コード**: `FormMain.cs` btnExportCsv_Click メソッド内

```csharp
// 現在の実装（問題あり）
StreamWriter writer = new StreamWriter(filePath);

// ヘッダー行
writer.WriteLine("顧客ID,顧客名,年齢,電話番号,メールアドレス");

// データ行
foreach (var customer in customers)
{
    writer.WriteLine($"{customer.Id},{customer.Name},{customer.Age},{customer.Phone},{customer.Email}");
}

writer.Close();  // ← ここに到達する前に例外が発生するとCloseされない
```

**何が問題か**:
- `StreamWriter` が `using` ブロックで囲まれていない
- `writer.WriteLine()` の途中で例外が発生した場合、`writer.Close()` が実行されない
- ファイルハンドルが解放されず、ファイルがロックされたままになる（リソースリーク）
- ロックされたファイルは他のプログラムから開けなくなり、再起動が必要になることもある

**修正案**:
```csharp
// usingブロックでStreamWriterを囲む
using (var writer = new StreamWriter(filePath, false, new System.Text.UTF8Encoding(true)))
{
    // ヘッダー行
    writer.WriteLine("顧客ID,顧客名,年齢,電話番号,メールアドレス");

    // データ行
    foreach (var customer in customers)
    {
        writer.WriteLine($"{customer.Id},{customer.Name},{customer.Age},{customer.Phone},{customer.Email}");
    }
}
// usingブロックを抜けると自動的にClose/Disposeされる
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.cs: btnExportCsv_Click メソッド内 StreamWriterの使用箇所
【分類】バグ
【重要度】高
【内容】StreamWriterがusingブロックで囲まれていない。
        writer.WriteLine()の処理中に例外が発生した場合、writer.Close()が
        実行されず、ファイルハンドルが解放されない（リソースリーク）。
        ファイルがロックされたままになり、再実行やファイル削除ができなくなる。
【修正案】StreamWriterをusingブロックで囲み、例外発生時でも確実に
        リソースが解放されるよう修正してください。
```

### 問題3: 日本語の文字化け（Encoding指定なし）

**該当コード**: `FormMain.cs` btnExportCsv_Click メソッド内

```csharp
// 現在の実装（問題あり）
StreamWriter writer = new StreamWriter(filePath);
// → Encodingが指定されていない（デフォルトはUTF-8だがBOMなし）
```

**何が問題か**:
- `StreamWriter` のデフォルトエンコーディングはUTF-8（BOMなし）
- Excelで「BOMなしUTF-8」のCSVを開くと、日本語が文字化けする
- ヘッダー行の「顧客ID」「顧客名」等や、データの「田中太郎」等がすべて文字化けする
- ExcelでCSVを正しく表示するには **UTF-8 BOM付き** で出力する必要がある

**BOM（Byte Order Mark）とは**:
- ファイルの先頭に付加される特殊なバイト列（EF BB BF）
- テキストエディタやExcelがファイルのエンコーディングを判別するための目印
- BOMがないと、ExcelはCSVをShift-JISとして解釈し、文字化けが発生する

**修正案**:
```csharp
// UTF-8 BOM付きでStreamWriterを作成
using (var writer = new StreamWriter(filePath, false, new System.Text.UTF8Encoding(true)))
{
    // UTF8Encoding(true) の true がBOMを付加する指定
    writer.WriteLine("顧客ID,顧客名,年齢,電話番号,メールアドレス");
    // ...
}
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.cs: btnExportCsv_Click メソッド内 StreamWriterの生成
【分類】バグ
【重要度】高
【内容】StreamWriterの生成時にEncodingが指定されていない。
        デフォルトのUTF-8（BOMなし）で出力されるため、ExcelでCSVを
        開いた際に日本語（ヘッダー「顧客ID,顧客名...」やデータ「田中太郎」等）が
        文字化けする。
        CSVファイルの利用者はExcelで開くことが想定されるため、
        UTF-8 BOM付きで出力する必要がある。
【修正案】StreamWriterのコンストラクタに new System.Text.UTF8Encoding(true) を
        指定し、BOM付きUTF-8で出力するよう変更してください。
        例: new StreamWriter(filePath, false, new System.Text.UTF8Encoding(true))
```

## 修正後の完全なコード

3つの問題をすべて修正した場合:

```csharp
/// <summary>
/// 顧客一覧をCSVファイルにエクスポートする
/// </summary>
private void btnExportCsv_Click(object sender, EventArgs e)
{
    if (customers.Count == 0)
    {
        MessageBox.Show("エクスポートする顧客データがありません", "警告",
            MessageBoxButtons.OK, MessageBoxIcon.Warning);
        return;
    }

    // 【修正1】SaveFileDialogで出力先を選択
    using (var dialog = new SaveFileDialog())
    {
        dialog.Filter = "CSVファイル (*.csv)|*.csv";
        dialog.FileName = "customers.csv";
        dialog.Title = "CSVエクスポート先を選択";

        if (dialog.ShowDialog() != DialogResult.OK)
            return;

        string filePath = dialog.FileName;

        try
        {
            // 【修正2】usingブロックで囲む + 【修正3】UTF-8 BOM付き
            using (var writer = new StreamWriter(filePath, false, new System.Text.UTF8Encoding(true)))
            {
                // ヘッダー行
                writer.WriteLine("顧客ID,顧客名,年齢,電話番号,メールアドレス");

                // データ行
                foreach (var customer in customers)
                {
                    writer.WriteLine($"{customer.Id},{customer.Name},{customer.Age},{customer.Phone},{customer.Email}");
                }
            }

            MessageBox.Show($"CSVファイルを出力しました\n{filePath}", "完了",
                MessageBoxButtons.OK, MessageBoxIcon.Information);
        }
        catch (Exception ex)
        {
            MessageBox.Show($"CSV出力中にエラーが発生しました: {ex.Message}", "エラー",
                MessageBoxButtons.OK, MessageBoxIcon.Error);
        }
    }
}
```

## レビューのコツ

### 1. diffを読む順序

1. まずPR説明で「何をしたか」を把握する
2. 追加された `using` 文を確認する（`System.IO` が追加されている → ファイル操作がある）
3. 追加されたメソッドを上から順に読む
4. 各処理で「このケースでエラーが起きたらどうなるか」を考える

### 2. よくある問題パターン

| パターン | 確認すること |
|---------|------------|
| ファイルパスのハードコーディング | 環境依存しないか？ユーザーが選択できるか？ |
| リソースの解放漏れ | using文で囲まれているか？finallyブロックがあるか？ |
| 文字エンコーディング | 日本語を含むファイルでBOM付きUTF-8か？ |
| エラーハンドリング | try-catchの範囲は適切か？エラーメッセージは適切か？ |
| ハードコーディングされた値 | 設定ファイルや定数に外出しすべきか？ |

### 3. 指摘の優先度

今回の3つの問題の優先度:

| 優先度 | 問題 | 理由 |
|--------|------|------|
| 高 | リソースリーク（using未使用） | ファイルロックが発生し、システム障害につながる |

## まとめ

PRレビューでは「正常に動作するか」だけでなく、以下の観点も重要:

1. **異常系**: エラーが発生した場合に安全か（リソースリーク）
2. **環境依存**: 他のPCやサーバーでも動作するか（ハードコーディング）
3. **日本語対応**: 日本語データが正しく扱えるか（エンコーディング）

これらはBSEとして特に注意すべき、日本のシステム開発で頻出する問題パターンです。
