# 演習4: Pull Requestレビュー演習

## 状況設定
オフショア開発チーム（ベトナム）の開発者から、顧客管理アプリに「CSVエクスポート機能」を追加するPull Requestが提出されました。
BSEであるあなたは、このPRをレビューし、問題点を指摘する必要があります。

## あなたのタスク
1. `PR例/PR説明.md` を読み、PRの内容を把握する
2. `PR例/diff_before.cs`（変更前）と `PR例/diff_after.cs`（変更後）を比較する
3. `PR例/diff.patch` でgit diff形式の差分を確認する
4. 問題点を見つけ、レビュー指摘を書く

## ヒント
- このPRには **3つの問題点** が含まれています
- レビューの観点: セキュリティ、リソース管理、文字エンコーディング
- `06_Git/レビューチェックリスト.md` を参照しながらレビューしてください

---

## PRレビューの手順

### Step 1: PRの説明を読む

`PR例/PR説明.md` を読み、以下を把握する:

- **何を変更したか**: どの機能が追加・変更されたか
- **なぜ変更したか**: 変更の背景・目的は何か
- **影響範囲**: 変更がどのファイルに及ぶか

### Step 2: 差分（Diff）を確認する

PRの差分を確認する方法:

**方法A: before/afterファイルの比較**
- `diff_before.cs` と `diff_after.cs` を見比べる
- Visual Studioで2つのファイルを開き、目視で比較する

**方法B: diff.patchファイルの確認**
- `diff.patch` はgit diff形式の出力
- `-`（赤）= 削除された行、`+`（緑）= 追加された行

**方法C: 実際のGitHub/Azure DevOpsでの確認（実務の場合）**
- PR画面の「Files changed」タブで差分を確認
- 行番号をクリックしてコメントを入力

### Step 3: チェックリストに沿ってレビュー

`06_Git/レビューチェックリスト.md` の各項目を確認する。
特に以下の観点を重点的に確認:

| 確認観点 | 確認内容 |
|---------|---------|
| セキュリティ | ハードコーディングされた値がないか |
| リソース管理 | ファイルやDB接続が適切に解放されているか |
| 文字エンコーディング | 日本語を含むファイルの文字化け対策がされているか |
| エラー処理 | 例外が適切にハンドリングされているか |
| 仕様準拠 | 要件を満たしているか |

### Step 4: 指摘を書く

問題を見つけたら、以下のフォーマットで指摘を書く:

```
【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致 / 改善提案 / セキュリティ
【重要度】高 / 中 / 低
【内容】○○の処理で△△となっているが、□□とすべきではないか。
【修正案】△△を□□に変更してください。
```
