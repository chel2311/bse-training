# 演習3: コンフリクト解消

## 状況設定
計算機アプリの `Calculator.cs` に対して、2人の開発者が同時に異なる変更を行いました。
mainブランチとfeatureブランチで同じ箇所が変更されたため、マージ時にコンフリクトが発生します。
この演習ではコンフリクトを意図的に発生させ、正しく解消する方法を学びます。

## 前提条件
- 演習1・2が完了していること
- ブランチの作成・切り替え・マージの基本操作を理解していること

---

## シナリオ

```
開発者A（あなた）: mainブランチで Divide メソッドのエラーメッセージを日本語に統一
開発者B:           featureブランチで Divide メソッドに引数チェックを追加

→ 同じ Divide メソッドを異なる内容に変更 → マージ時にコンフリクト発生
```

---

## 手順

### Step 1: 初期状態の確認

```bash
cd C:\work\git-training
git checkout main
git pull
```

現在の `Calculator.cs` の Divide メソッド:
```csharp
public int Divide(int a, int b)
{
    if (b == 0)
        throw new DivideByZeroException("0で割ることはできません");
    return a / b;
}
```

### Step 2: mainブランチで変更（開発者Aの作業）

mainブランチで Divide メソッドを変更する:

```csharp
/// <summary>
/// 除算（エラーメッセージを統一）
/// </summary>
public int Divide(int a, int b)
{
    if (b == 0)
    {
        throw new ArgumentException("エラー: 除数に0は指定できません。正の整数を入力してください。");
    }
    return a / b;
}
```

```bash
git add Calculator.cs
git commit -m "Divideメソッドのエラーメッセージを詳細化"
```

### Step 3: featureブランチを作成して変更（開発者Bの作業をシミュレート）

**重要**: Step 2のコミットの「1つ前」の状態からブランチを作る。

```bash
# 1つ前のコミットからブランチを作成
git checkout -b feature/divide-validation HEAD~1
```

featureブランチで Divide メソッドを異なる内容に変更する:

```csharp
/// <summary>
/// 除算（引数チェック強化）
/// </summary>
public int Divide(int a, int b)
{
    if (b == 0)
        throw new DivideByZeroException("0で割ることはできません");

    // 結果が整数にならない場合の警告
    if (a % b != 0)
    {
        Console.WriteLine($"警告: {a} / {b} の結果は切り捨てられます");
    }
    return a / b;
}
```

```bash
git add Calculator.cs
git commit -m "Divideメソッドに整数除算の警告を追加"
```

### Step 4: マージを試みる（コンフリクト発生）

```bash
# mainブランチに切り替え
git checkout main

# featureブランチをマージ
git merge feature/divide-validation
```

出力例:
```
Auto-merging Calculator.cs
CONFLICT (content): Merge conflict in Calculator.cs
Automatic merge failed; fix conflicts and then commit the result.
```

→ コンフリクトが発生した。

### Step 5: コンフリクトの状態を確認

```bash
git status
```

出力例:
```
On branch main
You have unmerged paths.
  (fix conflicts and run "git commit")
  (use "git merge --abort" to abort the merge)

Unmerged paths:
  (use "git add <file>..." to mark resolution)
        both modified:   Calculator.cs
```

Calculator.cs を開くと、コンフリクトマーカーが挿入されている:

### Step 6: コンフリクトを解消

両方の変更を取り入れて、正しいコードに修正する:
（`コンフリクト例/resolved.cs` を参照）

1. コンフリクトマーカー（`<<<<<<<`、`=======`、`>>>>>>>`）を削除する
2. 両方の変更内容を確認し、最終的なコードを決定する
3. この場合: エラーメッセージの詳細化（開発者A） + 整数除算の警告（開発者B）を両方採用

```csharp
/// <summary>
/// 除算（エラーメッセージ統一 + 引数チェック強化）
/// </summary>
public int Divide(int a, int b)
{
    if (b == 0)
    {
        throw new ArgumentException("エラー: 除数に0は指定できません。正の整数を入力してください。");
    }

    // 結果が整数にならない場合の警告
    if (a % b != 0)
    {
        Console.WriteLine($"警告: {a} / {b} の結果は切り捨てられます");
    }
    return a / b;
}
```

### Step 7: 解消をコミット

```bash
# 解消したファイルをステージング
git add Calculator.cs

# マージコミット
git commit -m "コンフリクト解消: Divideメソッドのエラーメッセージ詳細化と整数除算警告を統合"

# リモートに送信
git push
```

## Visual Studioでのコンフリクト解消

Visual Studioにはコンフリクト解消用のGUIがある:

1. マージ実行後、Git Changes ウィンドウにコンフリクトファイルが表示される
2. ファイルをクリックすると「マージ エディター」が開く
3. 3ペイン表示:
   - 左: 現在のブランチ（HEAD）の変更
   - 右: マージしようとしたブランチの変更
   - 下: 結果（ここを編集する）
4. チェックボックスで採用する変更を選択、または手動で下ペインを編集
5. 「マージの受入」ボタンで確定

---

## コンフリクトを予防するために

1. **作業前に必ず `git pull`** する（最新の状態で作業開始）
2. **ブランチをこまめにマージ** する（長期間放置しない）
3. **変更範囲を小さくする** （大きな変更はコンフリクトリスクが高い）
4. **チーム内で作業範囲を共有** する（同じファイルを同時に編集しない）
