# 練習問題2: クエリのバグを見つけよう

## 状況設定
オフショア開発チームから、受注管理データベースに対する各種帳票用クエリが納品されました。
BSEであるあなたは、要件に対してクエリが正しく動作するかレビューを行う必要があります。

## あなたのタスク
1. `設計仕様書/受注管理DB仕様書.md` を読む
2. `SampleData.sql` でテスト用データを確認する
3. `Queries.sql` の各クエリを読み、要件とのずれ（バグ）を見つける
4. レビュー指摘を書く

## ヒント
- このスクリプトには **3つのバグ** が仕込まれています
- 各クエリの要件コメントをよく読み、クエリが要件を満たすか確認してください
- JOIN、GROUP BY、WHERE/HAVINGに注目してください

## SSMSでの確認方法
1. SSMSでサーバーに接続
2. まず `CreateTables.sql`（問題1の正しい版）を実行してテーブルを作成
3. 次に `SampleData.sql` を実行してテストデータを投入
4. `Queries.sql` の各クエリを1つずつ実行して結果を確認
5. テストデータから期待される結果と実際の結果を比較する

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: 該当クエリ（要件番号）
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】要件では○○と定義されているが、クエリでは△△となっている。
【修正案】△△を□□に変更してください。
```
