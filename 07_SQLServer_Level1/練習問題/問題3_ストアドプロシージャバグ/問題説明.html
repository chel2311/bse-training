練習問題3: ストアドプロシージャのバグを見つけよう

状況設定

オフショア開発チームから、受注登録を行うストアドプロシージャが納品されました。
BSEであるあなたは、設計仕様書のビジネスルールと処理フローに基づいてコードレビューを行う必要があります。

あなたのタスク

  1. 設計仕様書/受注管理DB仕様書.md の「6. ストアドプロシージャ仕様」と「5. ビジネスルール」を読む
  2. sp_CreateOrder.sql のストアドプロシージャを読む
  3. 仕様書と実装の差異（バグ）を見つける
  4. レビュー指摘を書く

ヒント

  • このストアドプロシージャには 3つのバグ が仕込まれています
  • ビジネスルール（BR-001、BR-002、BR-003）の実装を重点的に確認してください
  • データの整合性を保つための仕組みが適切かも確認してください

SSMSでの確認方法

  1. SSMSでサーバーに接続
  2. テーブルとサンプルデータが投入済みの状態で sp_CreateOrder.sql を実行
  3. ストアドプロシージャを呼び出してテスト:
    ```sql
    -- テスト実行
    EXEC sp_CreateOrder @CustomerID = 1, @ProductID = 2, @Quantity = 3;

-- 結果確認
SELECT * FROM Orders;
SELECT * FROM OrderDetails;
SELECT * FROM Products WHERE ProductID = 2;
```
4. 在庫数やTotalAmountが正しく更新されているか確認

レビュー指摘の書き方

見つけたバグは以下のフォーマットで書いてみてください:

【指摘箇所】ファイル名: 該当処理
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。

回答

自分で考えた後、回答解説/問題3_回答.md で答え合わせしてください。
