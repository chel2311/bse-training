# 練習問題1: テーブル設計（DDL）のバグを見つけよう

## 状況設定
オフショア開発チームから「受注管理データベース」のテーブル作成スクリプト（DDL）が納品されました。
BSEであるあなたは、設計仕様書と照らし合わせてコードレビューを行う必要があります。

## あなたのタスク
1. `設計仕様書/受注管理DB仕様書.md` を読む
2. `CreateTables.sql` のDDL（CREATE TABLE文）を読む
3. 仕様書と実装の差異（バグ）を見つける
4. レビュー指摘を書く

## ヒント
- このスクリプトには **3つのバグ** が仕込まれています
- すべて「設計仕様書のテーブル設計・ビジネスルール」と実装が異なるバグです
- 各テーブルの制約（PRIMARY KEY、FOREIGN KEY、CHECK、データ型）を重点的に確認してください

## SSMSでの確認方法
1. SSMSでサーバーに接続
2. 新しいクエリウィンドウを開く
3. `CreateTables.sql` の内容を貼り付けてF5で実行
4. オブジェクトエクスプローラーでテーブルを展開し、キーや制約を確認
5. テーブルを右クリック → 「デザイン」で各カラムの型と制約を確認

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: 該当箇所
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
```
