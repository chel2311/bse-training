Bug #1
INNER JOINで注文なし顧客が消える
重要度: 高
Bug #2
GROUP BYがProductIDでカテゴリ別にならない
重要度: 高
Bug #3
WHERE句にSUM()を記述（エラー）
重要度: 高

バグ一覧（3件）

バグ1: 注文がない顧客が表示されない（INNER JOIN → LEFT JOIN）

該当要件: 要件1「顧客別の受注合計金額を表示」

すべての顧客について、受注合計金額を表示する。
注文がない顧客も「0円」として表示すること。

該当コード: Queries.sql 要件1のクエリ

-- 現在の実装（バグ）
SELECT
    c.CustomerID,
    c.Name           AS CustomerName,
    SUM(o.TotalAmount) AS TotalOrderAmount
FROM
    Customers c
    INNER JOIN Orders o ON c.CustomerID = o.CustomerID
    -- ↑ INNER JOINのため、注文がない高橋健一（CustomerID=5）が結果に含まれない
GROUP BY
    c.CustomerID, c.Name
ORDER BY
    c.Name;

期待される結果:
| CustomerID | CustomerName | TotalOrderAmount |
|------------|-------------|-----------------|
| 3 | 佐藤次郎 | 103000.00 |
| 2 | 鈴木花子 | 75000.00 |
| 5 | 高橋健一 | 0.00 | -- この行が消えている
| 1 | 田中太郎 | 183000.00 |
| 4 | 山田美咲 | 45000.00 |

修正コード:

SELECT
    c.CustomerID,
    c.Name                        AS CustomerName,
    ISNULL(SUM(o.TotalAmount), 0) AS TotalOrderAmount
FROM
    Customers c
    LEFT JOIN Orders o ON c.CustomerID = o.CustomerID
GROUP BY
    c.CustomerID, c.Name
ORDER BY
    c.Name;
INNER JOIN vs LEFT JOIN の動作の違い

レビュー指摘例:

【指摘箇所】Queries.sql: 要件1 顧客別受注合計クエリ
【分類】仕様不一致
【重要度】高
【内容】要件では「すべての顧客」「注文がない顧客も0円として表示」と定義されているが、
        実装ではINNER JOINを使用しているため、注文のない顧客が結果から除外される。
        テストデータでは高橋健一（CustomerID=5）に注文がなく、この顧客が表示されない。
【修正案】INNER JOINをLEFT JOINに変更し、SUM(o.TotalAmount)をISNULL()で囲んで
        NULL時に0を返すようにしてください。

バグ2: GROUP BYのカラム指定が誤り（ProductID → Category）

該当要件: 要件2「カテゴリ別の売上ランキングTOP3」

商品カテゴリ別の売上合計金額を集計し、売上の多い順にTOP3を表示する。
カテゴリ名と売上合計金額を表示すること。

該当コード: Queries.sql 要件2のクエリ

-- 現在の実装（バグ）
SELECT TOP 3
    p.Category,
    SUM(od.Subtotal) AS TotalSales
FROM
    OrderDetails od
    INNER JOIN Products p ON od.ProductID = p.ProductID
    INNER JOIN Orders o ON od.OrderID = o.OrderID
WHERE
    o.Status <> N'キャンセル'
GROUP BY
    p.ProductID     -- ← CategoryではなくProductIDでグルーピングしている
ORDER BY
    TotalSales DESC;

問題点:
- SELECT句に p.Category があるが、GROUP BY句に p.Category が含まれていない
- SQL Serverではこの状態でクエリを実行するとエラーになる:
Column 'Products.Category' is invalid in the select list because it is not contained in either an aggregate function or the GROUP BY clause.
- さらに、ProductIDでグルーピングすると「商品別」の集計になり、「カテゴリ別」にならない

修正コード:

SELECT TOP 3
    p.Category,
    SUM(od.Subtotal) AS TotalSales
FROM
    OrderDetails od
    INNER JOIN Products p ON od.ProductID = p.ProductID
    INNER JOIN Orders o ON od.OrderID = o.OrderID
WHERE
    o.Status <> N'キャンセル'
GROUP BY
    p.Category
ORDER BY
    TotalSales DESC;

期待される結果（修正後）:
| Category | TotalSales |
|----------|-----------|
| 電子機器 | 208000.00 |
| 家具 | 150000.00 |

レビュー指摘例:

【指摘箇所】Queries.sql: 要件2 カテゴリ別売上ランキングクエリ GROUP BY句
【分類】バグ
【重要度】高
【内容】要件では「カテゴリ別」の売上集計と定義されているが、
        GROUP BY句にp.Categoryではなくp.ProductIDが指定されている。
        この状態ではSELECT句のp.CategoryがGROUP BYに含まれず
        SQL Serverの実行時にエラーとなる。
        また、仮にSELECTからCategoryを外してもProductID別の集計になり、
        カテゴリ別の集計にならない。
【修正案】GROUP BY句を p.ProductID から p.Category に変更してください。

バグ3: 集計関数をWHERE句に記述（WHERE → HAVING）

該当要件: 要件3「今月の受注で合計10万円以上の顧客一覧」

顧客ごとの受注合計金額が10万円以上の顧客を一覧表示する。

該当コード: Queries.sql 要件3のクエリ

-- 現在の実装（バグ）
SELECT
    c.Name             AS CustomerName,
    SUM(o.TotalAmount) AS MonthlyTotal
FROM
    Orders o
    INNER JOIN Customers c ON o.CustomerID = c.CustomerID
WHERE
    o.OrderDate >= '2024-02-01'
    AND o.OrderDate < '2024-03-01'
    AND o.Status <> N'キャンセル'
    AND SUM(o.TotalAmount) >= 100000   -- ← WHERE句に集計関数は使用不可
GROUP BY
    c.CustomerID, c.Name
ORDER BY
    MonthlyTotal DESC;

問題点:
- WHERE句は「集計前の個々の行に対する条件」を指定する場所
- SUM()などの集計関数はWHERE句に使用できない
- SQL Serverでは以下のエラーになる:
An aggregate may not appear in the WHERE clause unless it is in a subquery contained in a HAVING clause.
- 集計結果に対する条件はHAVING句に記述する必要がある

修正コード:

SELECT
    c.Name             AS CustomerName,
    SUM(o.TotalAmount) AS MonthlyTotal
FROM
    Orders o
    INNER JOIN Customers c ON o.CustomerID = c.CustomerID
WHERE
    o.OrderDate >= '2024-02-01'
    AND o.OrderDate < '2024-03-01'
    AND o.Status <> N'キャンセル'
GROUP BY
    c.CustomerID, c.Name
HAVING
    SUM(o.TotalAmount) >= 100000
ORDER BY
    MonthlyTotal DESC;

期待される結果（修正後）:
| CustomerName | MonthlyTotal |
|-------------|-------------|
| 佐藤次郎 | 103000.00 |

SQL実行順序と WHERE / HAVING の役割
FROM / JOIN（テーブル結合）
↓
WHERE（行のフィルタ: 日付、ステータス等）
↓
GROUP BY（グループ化）
↓
HAVING（集計結果のフィルタ: SUM >= 100000）
↓
SELECT（結果の出力）
↓
ORDER BY（並べ替え）
× WHERE句にSUM()を書くとエラー: WHERE はGROUP BY前に実行されるため、集計関数は使用不可

レビュー指摘例:

【指摘箇所】Queries.sql: 要件3 今月の受注10万円以上クエリ WHERE句
【分類】バグ
【重要度】高
【内容】要件の「合計10万円以上」という条件を実装するために
        WHERE句にSUM(o.TotalAmount) >= 100000と記述しているが、
        WHERE句に集計関数は使用できずSQL Serverの実行時にエラーとなる。
        集計結果に対する条件はHAVING句に記述する必要がある。
        WHERE句は集計前の個々の行に対する条件（日付やステータス）、
        HAVING句は集計後のグループに対する条件（合計金額）と使い分ける。
【修正案】WHERE句から AND SUM(o.TotalAmount) >= 100000 を削除し、
        GROUP BY句の後に HAVING SUM(o.TotalAmount) >= 100000 を追加してください。

バグの見つけ方のコツ

  1. 「すべて」「全件」という要件にはLEFT JOINを疑う
    - INNER JOINは一致するレコードのみ返す
    - 「すべての顧客」「注文のない顧客も含む」などの要件はLEFT JOINが必須
    - バグ1はこの観点で発見できる

  2. SELECT句とGROUP BY句の整合性を確認する
    - SELECT句に集計関数でないカラムがある場合、GROUP BY句にそのカラムが必要
    - 「何の単位で集計するか」が要件と一致しているか確認
    - バグ2はこの観点で発見できる

  3. WHERE句とHAVING句の使い分けを確認する
    - WHERE: 集計前の行フィルタ（個々のレコードに対する条件）
    - HAVING: 集計後のグループフィルタ（集計結果に対する条件）
    - 「合計が○○以上」「件数が○○以上」はHAVINGで書くべき条件
    - バグ3はこの観点で発見できる

  4. SSMSで実行してエラーメッセージを読む
    - バグ2とバグ3はSQL実行時にエラーになる
    - エラーメッセージを読めば原因が分かることが多い
    - バグ1は実行は成功するが結果が間違っている（サイレントバグ）ため注意
