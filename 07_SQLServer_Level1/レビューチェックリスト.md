# SQL Server コードレビューチェックリスト（BSE向け）

## 使い方
SQLレビュー時に以下の項目を1つずつ確認していく。
各項目を確認したら「OK」「NG」「N/A（該当なし）」を記入する。

---

## 1. テーブル設計（DDL）

| No | 確認項目 | 結果 |
|----|---------|------|
| 1-1 | すべてのテーブルに主キー（PRIMARY KEY）が定義されているか | |
| 1-2 | テーブル間の参照関係に外部キー制約（FOREIGN KEY）が設定されているか | |
| 1-3 | 必須カラムにNOT NULL制約が設定されているか | |
| 1-4 | 値の範囲制限が必要なカラムにCHECK制約が設定されているか | |
| 1-5 | デフォルト値が必要なカラムにDEFAULT制約が設定されているか | |
| 1-6 | カラムのデータ型は設計仕様書通りか（特にDECIMAL/INT/NVARCHARの区別） | |
| 1-7 | 金額系カラムにDECIMAL型が使用されているか（INTやFLOATはNG） | |
| 1-8 | 日本語文字列にNVARCHAR型が使用されているか（VARCHARはNG） | |
| 1-9 | 自動採番（IDENTITY）の設定は仕様通りか | |

## 2. クエリ（SELECT）

| No | 確認項目 | 結果 |
|----|---------|------|
| 2-1 | JOINの種類は要件に合っているか（INNER JOIN / LEFT JOIN） | |
| 2-2 | 「すべて表示」の要件にINNER JOINを使っていないか | |
| 2-3 | JOIN条件（ON句）のカラムは正しいか | |
| 2-4 | SELECT句の集計関数以外のカラムがすべてGROUP BY句に含まれているか | |
| 2-5 | 集計結果への条件がWHERE句ではなくHAVING句に記述されているか | |
| 2-6 | WHERE句の条件は要件通りか（比較演算子、範囲、LIKE等） | |
| 2-7 | NULLの比較に `IS NULL` / `IS NOT NULL` を使っているか（`= NULL` はNG） | |
| 2-8 | 不要な `SELECT *` がないか（必要なカラムのみ指定すべき） | |

## 3. データ更新（INSERT / UPDATE / DELETE）

| No | 確認項目 | 結果 |
|----|---------|------|
| 3-1 | INSERT時にNOT NULLカラムに値が指定されているか | |
| 3-2 | UPDATE/DELETEにWHERE句があるか（WHERE句なし=全件更新/削除） | |
| 3-3 | 日本語文字列にNプレフィックスが付いているか（N'文字列'） | |
| 3-4 | 外部キー制約に違反するINSERT/UPDATE/DELETEがないか | |

## 4. ストアドプロシージャ

| No | 確認項目 | 結果 |
|----|---------|------|
| 4-1 | 複数テーブルの更新がトランザクション内で実行されているか（BEGIN TRAN/COMMIT） | |
| 4-2 | エラー発生時にROLLBACKされるか（TRY-CATCHまたは@@ERROR） | |
| 4-3 | ビジネスルールが正しく実装されているか（仕様書と照合） | |
| 4-4 | 入力パラメータのバリデーション（NULL、範囲、存在チェック）があるか | |
| 4-5 | 処理フローの順序は仕様書通りか | |
| 4-6 | 関連テーブルの連動更新が漏れていないか（合計金額の再計算等） | |

## 5. パフォーマンス

| No | 確認項目 | 結果 |
|----|---------|------|
| 5-1 | 外部キーカラムにインデックスが作成されているか | |
| 5-2 | WHERE句で頻繁に検索されるカラムにインデックスがあるか | |
| 5-3 | LIKE検索で先頭ワイルドカード（'%xxx'）を使用していないか | |
| 5-4 | 不要なSELECT *（全カラム取得）がないか | |

## 6. セキュリティ

| No | 確認項目 | 結果 |
|----|---------|------|
| 6-1 | アプリケーションからのSQL実行でパラメータ化クエリを使用しているか | |
| 6-2 | 文字列結合でSQL文を組み立てていないか（SQLインジェクションの危険） | |
| 6-3 | 接続文字列にパスワードがハードコーディングされていないか | |

---

## 指摘テンプレート

```
【指摘箇所】ファイル名: 該当箇所（テーブル名、クエリ番号、行番号等）
【分類】バグ / 仕様不一致 / 改善提案
【重要度】高 / 中 / 低
【内容】○○の処理で△△となっているが、設計仕様書では□□と定義されている。
【修正案】△△を□□に変更してください。
```

### 重要度の基準

| 重要度 | 基準 | 例 |
|--------|------|----|
| 高 | データ不整合・エラー・ビジネスルール違反 | 外部キー欠落、トランザクション欠落、集計ミス |
| 中 | 要件との差異だが致命的ではない | 並び順の違い、表示フォーマットの差異 |
| 低 | 品質向上のための改善提案 | インデックス追加、命名規則の統一 |
