## 対象者

| No | テーマ | 内容 |
|----|--------|------|
| 0-1 | 変数と型 | int, string, bool, decimal等。型の違いが起こすバグ |
| 0-2 | 制御構造 | if/else, for, while, switch。条件分岐の読み方 |

| 0-6 | DB接続基礎 | SQL Server接続、SELECT/INSERT/UPDATE/DELETE |
| 0-7 | 例外処理 | try-catch、エラーハンドリングの基本 |

## Phase 1: C# WinForms（Level 1 - 単一バグ）
> GUI付き顧客管理アプリで「仕様との差異」を見つける訓練

| No | 問題テーマ | 仕込むバグ | 難易度 |
|----|-----------|-----------|--------|
| 1-1 | 顧客追加が反映されない | イベントハンドラ未接続 | ★☆☆ |
| 1-2 | 検索が一致しない | 大文字小文字の未考慮 | ★☆☆ |
| 1-3 | 削除確認が出ない | 確認ダイアログ未実装 | ★☆☆ |
| 1-4 | 不正な値が入力できる | 入力バリデーション不足 | ★★☆ |
| 1-5 | 金額計算が合わない | 税込計算ロジックミス | ★★☆ |
| 1-6 | DB接続エラー | 接続文字列の誤り/例外未処理 | ★★☆ |
| 1-7 | データが保存されない | INSERT文のパラメータミス | ★★☆ |

### 各問題のセット内容
1. **設計仕様書** - アプリの要件・画面設計・処理フロー
2. **バグ入りソースコード** - Visual Studioで開いてデバッグ可能
3. **レビューチェックリスト** - 何を確認すべきかのガイド
4. **回答・解説** - バグ箇所、原因、修正コード、レビュー指摘例文

## 学習の進め方
1. まず **Phase 0** で基礎概念を理解
2. **Phase 1** の問題1から順にバグ探し
3. 各問題で「設計仕様書を読む → コードを読む → バグを見つける → 修正案を書く」のサイクルを回す
4. 回答解説で答え合わせ、レビュー指摘例文を参考に指摘の書き方を学ぶ
5. Level 1をクリアしたらLevel 2へ
6. C#が終わったら他の言語へ横展開

## レビュー指摘の書き方テンプレート

```
【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致 / 改善提案
【重要度】高 / 中 / 低
【内容】○○の処理で△△となっているが、設計仕様書では□□と定義されている。
【修正案】△△を□□に変更してください。
```
