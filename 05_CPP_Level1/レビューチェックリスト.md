# C++ / CAD コードレビューチェックリスト（BSE向け）

## 使い方
コードレビュー時に以下の項目を1つずつ確認していく。
各項目を確認したら「OK」「NG」「N/A（該当なし）」を記入する。

---

## 1. 仕様との整合性

| No | 確認項目 | 結果 |
|----|---------|------|
| 1-1 | クラス構成（継承関係）は設計書通りか | |
| 1-2 | 各メソッドの戻り値の型・引数は仕様書通りか | |
| 1-3 | 計算式（面積、周長、座標等）は仕様書の数式と一致しているか | |
| 1-4 | エラーメッセージの文言は仕様書通りか | |
| 1-5 | 入力バリデーションは仕様書の条件を全パラメータ分満たしているか | |

## 2. メモリ管理（C++最重要項目）

| No | 確認項目 | 結果 |
|----|---------|------|
| 2-1 | すべての `new` に対応する `delete` があるか | |
| 2-2 | 配列の `new[]` に対応する `delete[]` があるか | |
| 2-3 | デストラクタで保持するポインタを適切に解放しているか | |
| 2-4 | 基底クラスのデストラクタが `virtual` になっているか | |
| 2-5 | `delete` 後にポインタを `nullptr` に設定しているか | |
| 2-6 | 例外発生時にメモリリークが起きないか | |

## 3. ポインタ安全性

| No | 確認項目 | 結果 |
|----|---------|------|
| 3-1 | ポインタの `nullptr` チェックがあるか | |
| 3-2 | `delete` 済みのポインタを使用していないか（ダングリングポインタ） | |
| 3-3 | 配列の範囲外アクセスがないか | |
| 3-4 | 関数の戻り値ポインタの `nullptr` チェックがあるか | |

## 4. 計算精度

| No | 確認項目 | 結果 |
|----|---------|------|
| 4-1 | 座標・寸法に `double` を使用しているか（`float` ではないか） | |
| 4-2 | 定数（円周率等）が適切な精度で定義されているか | |
| 4-3 | 整数除算による精度落ちがないか（`int / int` ではなく `double` で計算） | |
| 4-4 | 0除算が発生しないか | |
| 4-5 | 数学関数（sqrt, sin, cos等）の `#include <cmath>` が含まれているか | |

## 5. クラス設計

| No | 確認項目 | 結果 |
|----|---------|------|
| 5-1 | 純粋仮想関数がすべての派生クラスで実装されているか | |
| 5-2 | `override` キーワードが付いているか | |
| 5-3 | アクセス修飾子（public, protected, private）が適切か | |
| 5-4 | ヘッダファイル（.h）の宣言と実装ファイル（.cpp）の定義が一致しているか | |
| 5-5 | `#pragma once` がヘッダファイルに含まれているか | |

## 6. コンストラクタ・バリデーション

| No | 確認項目 | 結果 |
|----|---------|------|
| 6-1 | コンストラクタで全パラメータの入力値を検証しているか | |
| 6-2 | 不正な値に対して適切な例外（std::invalid_argument等）を送出しているか | |
| 6-3 | メンバ変数が未初期化のまま使われていないか | |

## 7. 例外処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 7-1 | main関数でtry-catchにより例外を捕捉しているか | |
| 7-2 | 例外発生時にユーザーにメッセージが表示されるか | |
| 7-3 | 空のcatchブロック（例外の握りつぶし）がないか | |
| 7-4 | 例外送出時にメモリリークが起きないか | |

## 8. ファイル構成

| No | 確認項目 | 結果 |
|----|---------|------|
| 8-1 | ヘッダファイルに実装コードを書いていないか（インライン関数除く） | |
| 8-2 | 必要な `#include` が漏れなく記載されているか | |
| 8-3 | ヘッダファイルで `using namespace std;` を使っていないか | |
| 8-4 | .vcxproj にすべてのソースファイルが含まれているか | |

## C++レビューの優先度ガイド

**必ず確認（C#にはない観点）**:
- メモリリーク（new/delete の対応）
- ポインタの安全性（nullptrチェック）
- デストラクタの実装

**C#と共通の観点**:
- 計算式の正確性
- バリデーションの網羅性
- 仕様書との整合性
