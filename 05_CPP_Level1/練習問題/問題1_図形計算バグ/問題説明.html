練習問題1: 図形計算モジュールのバグを見つけよう

状況設定

CADソフト開発プロジェクトで、オフショア開発チームから図形計算モジュールの初期実装が納品されました。
BSEであるあなたは、設計仕様書と照らし合わせてコードレビューを行う必要があります。

このモジュールはC++で書かれていますが、Visual Studioがなくてもテキストエディタでソースコードを読めます。

あなたのタスク

  1. 設計仕様書/図形描画ツール仕様書.md を読む
  2. ShapeCalc/ フォルダのソースコードを読む
  3. 仕様書と実装の差異（バグ）を見つける
  4. レビュー指摘を書く

ヒント

  • このコードには 3つのバグ が仕込まれています
  • バグの種類: 計算ミス、メモリ管理、バリデーション不足
  • ヘッダファイル（.h）と実装ファイル（.cpp）の両方を確認してください
  • 特に仕様書の太字部分と実装を見比べてください

C++コードの読み方ガイド

C++が初めての方は以下を参考にしてください:

C++の書き方 意味 C#での対応
#include "Shape.h" ヘッダファイルの読み込み using に近い
class Circle : public Shape Shapeを継承したCircleクラス class Circle : Shape
virtual double GetArea() = 0; 抽象メソッド（純粋仮想関数） abstract double GetArea();
double GetArea() override; オーバーライドの宣言 override double GetArea()
Shape* shape = new Circle(5.0); ヒープにオブジェクト生成 Shape shape = new Circle(5.0);
delete shape; メモリ解放 （不要。ガベージコレクタが自動回収）
std::vector<Shape*> ポインタのリスト List<Shape>
shape->GetArea() ポインタ経由のメソッド呼び出し shape.GetArea()
std::cout << value コンソール出力 Console.Write(value)

Visual Studioでの確認方法

  1. ShapeCalc.sln をダブルクリックしてVisual Studioで開く
  2. F5キーでデバッグ実行
  3. コンソールに表示される計算結果を仕様書の計算式と比較する
  4. 仕様と異なる動作をする箇所がバグです

レビュー指摘の書き方

見つけたバグは以下のフォーマットで書いてみてください:

【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
