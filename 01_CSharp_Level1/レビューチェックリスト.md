# C# WinForms コードレビューチェックリスト（BSE向け）

## 使い方
コードレビュー時に以下の項目を1つずつ確認していく。
各項目を確認したら「OK」「NG」「N/A（該当なし）」を記入する。

---

## 1. 仕様との整合性

| No | 確認項目 | 結果 |
|----|---------|------|
| 1-1 | 画面の構成要素（ボタン、テキストボックス等）は設計書通りか | |
| 1-2 | 各ボタンの動作は仕様書の機能仕様通りか | |
| 1-3 | エラーメッセージの文言は仕様書通りか | |
| 1-4 | 入力バリデーションは仕様書の条件を満たしているか | |
| 1-5 | 計算ロジック（合計、平均等）は仕様書通りか | |

## 2. イベント処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 2-1 | すべてのボタンにイベントハンドラが接続されているか（Designer.cs確認） | |
| 2-2 | フォームロード時の初期化処理は適切か | |
| 2-3 | 一覧選択時の処理は適切か | |

## 3. 入力チェック

| No | 確認項目 | 結果 |
|----|---------|------|
| 3-1 | 必須項目の空チェックがあるか | |
| 3-2 | 数値項目の型チェックがあるか | |
| 3-3 | 数値の範囲チェックがあるか（上限・下限） | |
| 3-4 | 文字列のフォーマットチェックがあるか（メール、電話） | |

## 4. エラー処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 4-1 | DB接続時にtry-catchがあるか | |
| 4-2 | エラー時にユーザーにメッセージが表示されるか | |
| 4-3 | 空のcatchブロック（エラーの握りつぶし）がないか | |

## 5. UI操作

| No | 確認項目 | 結果 |
|----|---------|------|
| 5-1 | 削除等の重要操作前に確認ダイアログがあるか | |
| 5-2 | 操作後に画面表示が更新されるか | |
| 5-3 | 操作後に入力欄がクリアされるか（仕様で指定がある場合） | |
| 5-4 | 一覧未選択時に適切な警告が出るか | |

## 6. データ処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 6-1 | 金額計算に適切な型（decimal）を使っているか | |
| 6-2 | 0除算が発生しないか（平均計算時等） | |
| 6-3 | NULL値の考慮がされているか | |

## 7. セキュリティ

| No | 確認項目 | 結果 |
|----|---------|------|
| 7-1 | SQLにパラメータを使っているか（文字列結合でのSQL組み立てはNG） | |
| 7-2 | 接続文字列にパスワードがハードコーディングされていないか | |

---

