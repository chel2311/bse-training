バグ一覧（3件）

バグ1: 年齢の範囲チェックがない

該当仕様: FR-005 入力バリデーション

年齢: 必須。0以上150以下の整数。範囲外はエラー「年齢は0～150の整数で入力してください」

該当コード: FormMain.cs 38行目付近

// 現在の実装（バグ）
if (!int.TryParse(txtAge.Text, out int age))
{
    MessageBox.Show("年齢は整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning);
    return;
}
// → 整数かどうかのチェックだけ。-100や999も通ってしまう

修正コード:

if (!int.TryParse(txtAge.Text, out int age) || age < 0 || age > 150)
{
    MessageBox.Show("年齢は0～150の整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning);
    return;
}

レビュー指摘例:

【指摘箇所】FormMain.cs: btnAdd_Click メソッド内 年齢チェック処理
【分類】仕様不一致
【重要度】高
【内容】設計仕様書 FR-005 では年齢は「0以上150以下の整数」と定義されているが、
        実装では整数型チェックのみで範囲チェックが行われていない。
        マイナス値や150を超える値が登録可能な状態。
【修正案】int.TryParse成功後に age < 0 || age > 150 の条件を追加し、
        範囲外の場合は「年齢は0～150の整数で入力してください」のエラーメッセージを表示してください。

バグ2: 追加後に入力欄がクリアされない

該当仕様: FR-001 顧客追加

追加後、入力欄をクリアする

該当コード: FormMain.cs 50行目付近

// 現在の実装（バグ）
customers.Add(customer);
// ← ここに入力欄のクリア処理がない
RefreshGrid(customers);

修正コード:

customers.Add(customer);

// 入力欄をクリア
txtName.Text = "";
txtPhone.Text = "";
txtAge.Text = "";
txtEmail.Text = "";

RefreshGrid(customers);

レビュー指摘例:

【指摘箇所】FormMain.cs: btnAdd_Click メソッド内 顧客追加後の処理
【分類】仕様不一致
【重要度】中
【内容】設計仕様書 FR-001 では「追加後、入力欄をクリアする」と定義されているが、
        実装では追加後の入力欄クリア処理が行われていない。
        連続追加時に前の入力値が残り、誤って同じデータを複数登録してしまう危険がある。
【修正案】customers.Add(customer) の後に、txtName, txtPhone, txtAge, txtEmail の
        Text プロパティを空文字に設定するクリア処理を追加してください。

バグ3: 削除時に確認ダイアログが表示されない

該当仕様: FR-003 顧客削除

削除前に「○○を削除しますか？」と確認ダイアログを表示する
「はい」で削除実行、「いいえ」でキャンセル

該当コード: FormMain.cs 82行目付近

// 現在の実装（バグ）
if (customer != null)
{
    customers.Remove(customer);  // ← 確認なしで即削除
    RefreshGrid(customers);
}

修正コード:

if (customer != null)
{
    var result = MessageBox.Show(
        $"{customer.Name}を削除しますか？",
        "削除確認",
        MessageBoxButtons.YesNo,
        MessageBoxIcon.Question);

    if (result == DialogResult.Yes)
    {
        customers.Remove(customer);
        RefreshGrid(customers);
    }
}

レビュー指摘例:

【指摘箇所】FormMain.cs: btnDelete_Click メソッド内 削除処理
【分類】仕様不一致
【重要度】高
【内容】設計仕様書 FR-003 では削除前に「○○を削除しますか？」と
        確認ダイアログを表示すると定義されているが、
        実装では確認なしで即座に削除が実行される。
        ユーザーの誤操作によるデータ消失の危険がある。
【修正案】customers.Remove の前に MessageBox.Show で確認ダイアログを表示し、
        DialogResult.Yes の場合のみ削除を実行するよう修正してください。

バグの見つけ方のコツ

  1. 仕様書の太字や強調部分は特に注意して確認する
    - 今回のバグはすべて仕様書で太字になっていた箇所

  2. 実際に動かして確認する（デバッグ実行）
    - バグ1: 年齢に「-10」や「200」を入力して追加ボタンを押してみる
    - バグ2: データを追加した後、入力欄が空になるか確認
    - バグ3: 削除ボタンを押して確認ダイアログが出るか確認

  3. 「ユーザーが間違った操作をしたらどうなるか」を考える
    - バリデーションの不足はBSEレビューで最も多い指摘項目
