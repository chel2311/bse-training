### バグ1: 年齢の範囲チェックがない

**該当仕様**: FR-005 入力バリデーション
> 年齢: 必須。**0以上150以下の整数**。範囲外はエラー「年齢は0～150の整数で入力してください」

**該当コード**: `FormMain.cs` 38行目付近

```csharp
// 現在の実装（バグ）
if (!int.TryParse(txtAge.Text, out int age))
{
    MessageBox.Show("年齢は整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning);
    return;
}
// → 整数かどうかのチェックだけ。-100や999も通ってしまう
```

**修正コード**:
```csharp
if (!int.TryParse(txtAge.Text, out int age) || age < 0 || age > 150)
{
    MessageBox.Show("年齢は0～150の整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning);
    return;
}
```

        マイナス値や150を超える値が登録可能な状態。

        範囲外の場合は「年齢は0～150の整数で入力してください」のエラーメッセージを表示してください。

### バグ3: 削除時に確認ダイアログが表示されない

**該当仕様**: FR-003 顧客削除
> **削除前に「○○を削除しますか？」と確認ダイアログを表示する**
> 「はい」で削除実行、「いいえ」でキャンセル

**該当コード**: `FormMain.cs` 82行目付近

```csharp
// 現在の実装（バグ）
if (customer != null)
{
    customers.Remove(customer);  // ← 確認なしで即削除
    RefreshGrid(customers);
}
```

**修正コード**:
```csharp
if (customer != null)
{
    var result = MessageBox.Show(
        $"{customer.Name}を削除しますか？",
        "削除確認",
        MessageBoxButtons.YesNo,
        MessageBoxIcon.Question);

    if (result == DialogResult.Yes)
    {
        customers.Remove(customer);
        RefreshGrid(customers);
    }
}
```

## バグの見つけ方のコツ

1. **仕様書の太字や強調部分は特に注意して確認する**
   - 今回のバグはすべて仕様書で太字になっていた箇所

2. **実際に動かして確認する（デバッグ実行）**
   - バグ1: 年齢に「-10」や「200」を入力して追加ボタンを押してみる
   - バグ2: データを追加した後、入力欄が空になるか確認
   - バグ3: 削除ボタンを押して確認ダイアログが出るか確認

3. **「ユーザーが間違った操作をしたらどうなるか」を考える**
   - バリデーションの不足はBSEレビューで最も多い指摘項目
