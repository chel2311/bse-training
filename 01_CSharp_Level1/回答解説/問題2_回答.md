### バグ1: 検索が大文字・小文字を区別してしまう

**該当仕様**: FR-004 検索
> **大文字・小文字を区別しない**

**該当コード**: `FormMain.cs` btnSearch_Click メソッド内

```csharp
// 現在の実装（バグ）
var results = customers.FindAll(c =>
    c.Name.Contains(keyword) ||
    c.Email.Contains(keyword));
// → Contains() に StringComparison が指定されていないため、
//   大文字・小文字を区別してしまう
//   例: "tanaka" で検索すると "Tanaka@example.com" がヒットしない
```

**修正コード**:
```csharp
var results = customers.FindAll(c =>
    c.Name.Contains(keyword, StringComparison.OrdinalIgnoreCase) ||
    c.Email.Contains(keyword, StringComparison.OrdinalIgnoreCase));
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.cs: btnSearch_Click メソッド内 検索処理
【分類】仕様不一致
【重要度】高
【内容】設計仕様書 FR-004 では「大文字・小文字を区別しない」と定義されているが、
        実装では Contains() に StringComparison.OrdinalIgnoreCase が指定されておらず、
        大文字・小文字を区別した検索になっている。
        例えばメールアドレスが "Tanaka@example.com" の場合、
        "tanaka" で検索してもヒットしない。
【修正案】Contains() の第2引数に StringComparison.OrdinalIgnoreCase を指定してください。
```

### バグ2: 検索結果0件でメッセージが表示されない

**該当仕様**: FR-004 検索
> 検索結果が0件の場合**「該当する顧客はありません」**と表示

**該当コード**: `FormMain.cs` btnSearch_Click メソッド内

```csharp
// 現在の実装（バグ）
var results = customers.FindAll(c =>
    c.Name.Contains(keyword) ||
    c.Email.Contains(keyword));

// ← ここに 0件チェックとメッセージ表示がない

RefreshGrid(results);
// → 検索結果が0件でも何もメッセージが出ず、空の一覧が表示されるだけ
//   ユーザーは検索が正しく動作しているのか分からない
```

**修正コード**:
```csharp
var results = customers.FindAll(c =>
    c.Name.Contains(keyword, StringComparison.OrdinalIgnoreCase) ||
    c.Email.Contains(keyword, StringComparison.OrdinalIgnoreCase));

if (results.Count == 0)
{
    MessageBox.Show("該当する顧客はありません", "検索結果", MessageBoxButtons.OK, MessageBoxIcon.Information);
}

RefreshGrid(results);
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.cs: btnSearch_Click メソッド内 検索結果チェック処理
【分類】仕様不一致
【重要度】中
【内容】設計仕様書 FR-004 では「検索結果が0件の場合は該当する顧客はありませんと表示」
        と定義されているが、実装では0件チェックとメッセージ表示の処理がない。
        検索結果が0件の場合、空の一覧が表示されるだけでユーザーへのフィードバックがない。
【修正案】RefreshGrid の前に results.Count == 0 の判定を追加し、
        0件の場合は MessageBox.Show で「該当する顧客はありません」を表示してください。
```

### バグ3: ステータスの平均年齢が整数になる（小数が切り捨てられる）

**該当仕様**: FR-006 ステータス表示
> 全顧客の年齢の平均（**小数第1位まで**）

**該当コード**: `FormMain.cs` UpdateStatus メソッド内

```csharp
// 現在の実装（バグ）
int avgAge = count > 0 ? totalAge / count : 0;
lblStatus.Text = $"件数: {count}件  合計年齢: {totalAge}  平均年齢: {avgAge}";
// → int同士の除算のため、小数部分が切り捨てられる
//   例: totalAge=63, count=2 の場合、63/2=31（正しくは31.5）
//   例: totalAge=105, count=3 の場合、105/3=35（たまたま正しいが偶然）
```

**修正コード**:
```csharp
double avgAge = count > 0 ? (double)totalAge / count : 0;
lblStatus.Text = $"件数: {count}件  合計年齢: {totalAge}  平均年齢: {avgAge:F1}";
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.cs: UpdateStatus メソッド内 平均年齢計算処理
【分類】仕様不一致
【重要度】高
【内容】設計仕様書 FR-006 では平均年齢は「小数第1位まで」表示すると定義されているが、
        実装では int 型同士の除算（totalAge / count）を行っているため、
        小数部分が切り捨てられている。
        例: 合計年齢63、件数2の場合、63/2=31 と表示される（正しくは31.5）。
        また、表示フォーマットにも小数第1位の書式指定がない。
【修正案】avgAge の型を double に変更し、除算時にキャスト ((double)totalAge / count) を行い、
        文字列フォーマットに :F1 を指定して小数第1位まで表示してください。
```

## バグの見つけ方のコツ

1. **仕様書の太字や強調部分は特に注意して確認する**
   - 今回のバグはすべて仕様書で太字になっていた箇所

2. **実際に動かして確認する（デバッグ実行）**
   - バグ1: 検索欄に "Tanaka" と大文字で入力して検索してみる（メールアドレスの一部）
   - バグ2: 存在しない名前やアドレス（例: "xyz"）で検索してみる
   - バグ3: 顧客を1件削除して2件にし、ステータスバーの平均年齢を電卓で検算する

3. **型に注意する（C#の落とし穴）**
   - int / int は結果もint（小数が切り捨てられる）
   - 小数を含む計算が必要な場合は double や decimal にキャストする
   - これはC#初学者が最も引っかかりやすいバグの一つ

4. **文字列比較のデフォルト動作を理解する**
   - C#のContains()はデフォルトで大文字・小文字を区別する
   - 仕様で「区別しない」と書かれていたらStringComparisonの指定が必要
