# 顧客管理アプリケーション（DB版） 設計仕様書

## 1. 概要

顧客情報の登録・表示・検索・削除を行うWindowsデスクトップアプリケーション。
データストアとしてSQL Serverを使用し、永続化されたデータに対してCRUD操作を行う。

## 2. 開発環境

| 項目 | 仕様 |
|------|------|
| 言語 | C# |
| フレームワーク | .NET 8 |
| UI | Windows Forms |
| DB | SQL Server (localhost) |
| ORM | なし（ADO.NET直接使用） |
| IDE | Visual Studio 2022 |

## 3. データベース仕様

### 3.1 接続情報

| 項目 | 値 |
|------|------|
| サーバー | localhost |
| データベース名 | CustomerDB |
| 認証方式 | Windows認証 |

### 3.2 接続文字列管理

- **接続文字列は app.config で管理すること（ハードコーディング禁止）**
- app.config の `<connectionStrings>` セクションに定義する
- コード側は `ConfigurationManager.ConnectionStrings` で取得する

### 3.3 テーブル定義

#### Customers テーブル

| カラム名 | 型 | 制約 | 説明 |
|---------|------|------|------|
| Id | INT | PRIMARY KEY, IDENTITY(1,1) | 顧客ID（自動採番） |
| Name | NVARCHAR(100) | NOT NULL | 氏名 |
| Phone | NVARCHAR(20) | NULL許可 | 電話番号 |
| Age | INT | NOT NULL | 年齢 |
| Email | NVARCHAR(200) | NULL許可 | メールアドレス |

### 3.4 SQL実装規約

- **SQLはパラメータ化クエリを使用すること（SQLインジェクション対策）**
  - 文字列結合によるSQL構築は禁止
  - `SqlCommand.Parameters.AddWithValue()` でパラメータを設定する
- **SqlConnection, SqlCommand, SqlDataReader は必ず using で囲むこと**
  - リソースリーク防止のため、IDisposable を実装するオブジェクトは確実に Dispose する
- **DB接続時は必ず try-catch で例外処理を行い、ユーザーにエラーメッセージを表示すること**
  - `SqlException` をキャッチし、ユーザーにわかりやすいメッセージを表示する
  - アプリケーションがクラッシュしないよう例外を適切にハンドリングする

## 5. 機能仕様

### 5.1 顧客追加（FR-001）
- 「追加」ボタン押下で入力欄の値をデータベースに INSERT する
- IDはデータベースの IDENTITY で自動採番
- **追加後、入力欄をクリアする**
- **追加後、一覧を更新して新しい顧客が表示されること**
- ステータスバーの件数・集計を更新する

### 5.2 顧客編集（FR-002）
- 一覧で行を選択した状態で「編集」ボタン押下
- 選択行の値が入力欄に表示される
- 入力欄を修正後、再度「編集」ボタンで UPDATE 実行
- **一覧が選択されていない場合は「顧客を選択してください」と警告表示**

### 5.3 顧客削除（FR-003）
- 一覧で行を選択した状態で「削除」ボタン押下
- **削除前に「○○を削除しますか？」と確認ダイアログを表示する**
- 「はい」で DELETE 実行、「いいえ」でキャンセル
- **一覧が選択されていない場合は「顧客を選択してください」と警告表示**

### 5.4 検索（FR-004）
- 検索テキストボックスにキーワードを入力し「検索」ボタンで実行
- **氏名またはメールアドレスに対して部分一致検索を行う（SQL の LIKE を使用）**
- **大文字・小文字を区別しない**
- 検索結果が0件の場合「該当する顧客はありません」と表示
- 「全件表示」ボタンで検索をリセットし全データを表示

### 5.5 入力バリデーション（FR-005）
- 氏名: **必須。空欄の場合はエラー「氏名を入力してください」**
- 電話番号: 任意
- 年齢: **必須。0以上150以下の整数。範囲外はエラー「年齢は0～150の整数で入力してください」**
- メールアドレス: 任意

### 5.6 ステータス表示（FR-006）
- 画面下部に以下を常時表示:
  - 現在の表示件数
  - **全顧客の年齢の合計**
  - **全顧客の年齢の平均（小数第1位まで）**
- 追加・削除・検索時に自動更新される

## 6. アーキテクチャ

### 6.1 クラス構成

| クラス名 | 役割 |
|---------|------|
| Program | エントリポイント |
| Customer | 顧客データを保持するモデルクラス |
| DatabaseHelper | DB接続・CRUD操作を行うヘルパークラス |
| FormMain | UI制御・イベント処理 |

### 6.2 DatabaseHelper クラスの責務

- 接続文字列の取得（app.config から）
- 顧客の全件取得（SELECT）
- 顧客の追加（INSERT）
- 顧客の更新（UPDATE）
- 顧客の削除（DELETE）
- 顧客の検索（SELECT + LIKE）

## 7. 非機能要件

- **DB接続エラー時は適切なメッセージを表示し、アプリがクラッシュしないこと**
- 一覧の選択行は青色でハイライトされること
- **DB接続はusing文で確実にリソースを解放すること**
