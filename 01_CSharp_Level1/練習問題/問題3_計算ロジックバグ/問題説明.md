# 練習問題3: 注文管理アプリの計算ロジックバグを見つけよう

## 難易度: ★★☆ 中級

## 状況設定
オフショア開発チームから「注文管理システム」の初期実装が納品されました。
BSEであるあなたは、設計仕様書と照らし合わせてコードレビューを行う必要があります。

今回は計算ロジックに関するバグが中心です。
金額計算は顧客に直接影響するため、特に慎重なレビューが求められます。

## あなたのタスク
1. `設計仕様書.md` を読む
2. `OrderManager/` フォルダのソースコードを読む
3. 仕様書と実装の差異（バグ）を見つける
4. レビュー指摘を書く

## ヒント
- このコードには **3つのバグ** が仕込まれています
- すべて「設計仕様書の機能仕様」と実装が異なるバグです
- **計算ロジックを重点的にチェック** してください
- FormMain.cs の `btnAdd_Click`、`CalculateSubtotal`、`UpdateTotal` メソッドに注目

## Visual Studioでの確認方法
1. `OrderManager.sln` をダブルクリックしてVisual Studioで開く
2. F5キーでデバッグ実行
3. 設計仕様書の通りに操作してみる
4. 仕様と異なる動作をする箇所がバグです

### 動作確認の例
- 単価200円、数量5個、税率10%で追加 → 小計は1,100円になるはず
- 単価150円、数量3個、税率8%で追加 → 小計は486円になるはず
- 上記2件の合計は1,586円になるはず

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
```
