# VB6 レガシーコード レビューチェックリスト（BSE向け）

## 使い方
コードレビュー時に以下の項目を1つずつ確認していく。
各項目を確認したら「OK」「NG」「N/A（該当なし）」を記入する。

---

## 1. 仕様との整合性

| No | 確認項目 | 結果 |
|----|---------|------|
| 1-1 | 画面の構成要素（ボタン、テキストボックス等）は設計書通りか | |
| 1-2 | 各ボタンの動作は仕様書の機能仕様通りか | |
| 1-3 | エラーメッセージの文言は仕様書通りか | |
| 1-4 | 入力バリデーションは仕様書の条件を満たしているか | |
| 1-5 | 計算ロジック（勤続年数、金額等）は仕様書通りか | |
| 1-6 | 一覧表示のカラム構成と表示形式は仕様書通りか | |

## 2. VB6基本構造

| No | 確認項目 | 結果 |
|----|---------|------|
| 2-1 | すべてのモジュールで `Option Explicit` が宣言されているか | |
| 2-2 | `Variant` 型の不要な使用がないか | |
| 2-3 | `Integer` ではなく `Long` を使用しているか（オーバーフロー防止） | |
| 2-4 | ユーザー定義型（Type）の定義は標準モジュールに配置されているか | |

## 3. イベント処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 3-1 | イベントプロシージャ名が正しいか（コントロール名_イベント名） | |
| 3-2 | Form_Load で適切な初期化が行われているか | |
| 3-3 | ボタンクリック時の処理が仕様書の機能仕様に対応しているか | |

## 4. 入力チェック

| No | 確認項目 | 結果 |
|----|---------|------|
| 4-1 | 必須項目の空チェックがあるか | |
| 4-2 | 数値項目の型チェックがあるか（IsNumeric等） | |
| 4-3 | 日付項目のチェックに `IsDate` 関数を使っているか | |
| 4-4 | 重複チェック（一意制約）が実装されているか | |
| 4-5 | バリデーション失敗時に適切なエラーメッセージが表示されるか | |

## 5. エラー処理

| No | 確認項目 | 結果 |
|----|---------|------|
| 5-1 | 各プロシージャに `On Error GoTo` が設定されているか | |
| 5-2 | `On Error Resume Next`（エラー無視）が不適切に使われていないか | |
| 5-3 | エラーハンドラの後に `Exit Sub` / `Exit Function` があるか（正常時のフォールスルー防止） | |
| 5-4 | エラー時にユーザーにメッセージが表示されるか | |

## 6. UI操作

| No | 確認項目 | 結果 |
|----|---------|------|
| 6-1 | 削除等の重要操作前に確認ダイアログ（MsgBox vbYesNo）があるか | |
| 6-2 | 操作後に一覧表示が更新されるか | |
| 6-3 | 操作後に入力欄がクリアされるか（仕様で指定がある場合） | |
| 6-4 | 一覧未選択時に適切な警告が出るか | |
| 6-5 | SetFocus で適切なコントロールにフォーカスが移動するか | |

## 7. 関数・計算ロジック

| No | 確認項目 | 結果 |
|----|---------|------|
| 7-1 | `DateDiff` の引数順序（interval, date1, date2）は正しいか | |
| 7-2 | `DateAdd` の引数順序は正しいか | |
| 7-3 | `Format` の書式文字列は正しいか | |
| 7-4 | 0除算が発生しないか（平均計算時等） | |
| 7-5 | `CDate`, `CInt` 等の型変換エラーが考慮されているか | |

## 8. メモリ・リソース管理

| No | 確認項目 | 結果 |
|----|---------|------|
| 8-1 | オブジェクト変数に `Set obj = Nothing` で参照解放しているか | |
| 8-2 | `ReDim Preserve` の使用が適切か（パフォーマンスへの影響） | |
| 8-3 | DB接続（Connection/Recordset）の Close と Nothing が確実に行われているか | |
| 8-4 | ファイルを開いた場合、確実にClose/解放しているか | |

## 9. レガシーコード特有の注意点

| No | 確認項目 | 結果 |
|----|---------|------|
| 9-1 | 固定長文字列 `Dim s As String * 10` による切り捨てが発生していないか | |
| 9-2 | `ByRef`（デフォルト）で意図しない引数の変更が発生していないか | |
| 9-3 | 配列の下限（LBound）と上限（UBound）の取り扱いは正しいか | |
| 9-4 | 日付リテラル `#mm/dd/yyyy#` の月日の解釈は正しいか（米国式に注意） | |
| 9-5 | 2000年問題対応（2桁年号の解釈）は適切か | |
