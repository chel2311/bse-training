### バグ1: 勤続年数の計算が逆（マイナスの勤続年数が表示される）

**該当仕様**: FR-006 勤続年数計算
> **入社日から現在日までの年数をDateDiff関数で計算する**
> 計算式: `DateDiff("yyyy", 入社日, Now)` で年数を取得

**該当コード**: `modEmployee.bas` CalcYearsOfService関数

```vb
' 現在の実装（バグ）
Public Function CalcYearsOfService(dtHireDate As Date) As Long
    CalcYearsOfService = DateDiff("yyyy", Now, dtHireDate)
End Function
' → DateDiffの第2引数（開始日）と第3引数（終了日）が逆
' → 結果がマイナスになる（例: 2015年入社 → -11年）
```

**修正コード**:
```vb
Public Function CalcYearsOfService(dtHireDate As Date) As Long
    CalcYearsOfService = DateDiff("yyyy", dtHireDate, Now)
End Function
```

**DateDiff関数の構文**:
```
DateDiff(interval, date1, date2)
  interval: "yyyy"（年）, "m"（月）, "d"（日）
  date1: 開始日（古い日付）
  date2: 終了日（新しい日付）
  戻り値: date2 - date1 の差分
```

**レビュー指摘例**:
```

【分類】バグ
【重要度】高
【内容】設計仕様書 FR-006 では「DateDiff("yyyy", 入社日, Now)」で
        勤続年数を計算すると定義されているが、
        実装では DateDiff("yyyy", Now, dtHireDate) となっており、
        引数の順序（開始日と終了日）が逆になっている。
        結果としてマイナスの勤続年数が表示される。
【修正案】DateDiff("yyyy", Now, dtHireDate) を
        DateDiff("yyyy", dtHireDate, Now) に修正してください。
```

### バグ2: 社員番号の重複チェックがない

**該当仕様**: FR-001 社員追加
> **社員番号は重複不可。既に登録済みの社員番号の場合はエラー「この社員番号は既に登録されています」を表示する**

**該当コード**: `frmMain.frm` cmdAdd_Clickプロシージャ

```vb
' 現在の実装（バグ）
' 社員番号: 数字チェック
If Not IsValidEmpNo(Trim$(txtEmpNo.Text)) Then
    MsgBox "社員番号は数字で入力してください", vbExclamation, "入力エラー"
    txtEmpNo.SetFocus
    Exit Sub
End If

' ← ここに重複チェック処理がない

' 氏名: 必須チェック
If Len(Trim$(txtName.Text)) = 0 Then
' → 同じ社員番号で何人でも登録できてしまう
```

**修正コード**:
```vb
' 社員番号: 数字チェック
If Not IsValidEmpNo(Trim$(txtEmpNo.Text)) Then
    MsgBox "社員番号は数字で入力してください", vbExclamation, "入力エラー"
    txtEmpNo.SetFocus
    Exit Sub
End If

' 社員番号: 重複チェック
If FindEmployeeIndex(Trim$(txtEmpNo.Text)) > 0 Then
    MsgBox "この社員番号は既に登録されています", vbExclamation, "入力エラー"
    txtEmpNo.SetFocus
    Exit Sub
End If

' 氏名: 必須チェック
```

**ポイント**: `modEmployee.bas` に `FindEmployeeIndex` 関数が既に用意されている。この関数は社員番号で配列を検索し、見つかればインデックス（1以上）、見つからなければ0を返す。重複チェックに活用できる関数があるのに使われていないことに気づけるかがポイント。

**レビュー指摘例**:

### バグ3: 入社日のバリデーションでIsDate関数を使っていない

**該当仕様**: FR-004 入力バリデーション
> **IsDate関数を使用して日付形式チェックを行う。不正な場合はエラー「入社日を正しい日付形式(yyyy/mm/dd)で入力してください」**

**該当コード**: `frmMain.frm` cmdAdd_Clickプロシージャ

```vb
' 現在の実装（バグ）
If Len(txtHireDate.Text) <> 10 Then
    MsgBox "入社日を正しい日付形式(yyyy/mm/dd)で入力してください", vbExclamation, "入力エラー"
    txtHireDate.SetFocus
    Exit Sub
End If
' → 文字列の長さが10文字かどうかだけチェックしている
' → "aaaa/bb/cc" や "1234567890" のような不正な値が通ってしまう
```

**修正コード**:
```vb
If Not IsDate(txtHireDate.Text) Then
    MsgBox "入社日を正しい日付形式(yyyy/mm/dd)で入力してください", vbExclamation, "入力エラー"
    txtHireDate.SetFocus
    Exit Sub
End If
```

**IsDate関数について**:
```
IsDate(expression)
  引数: 判定対象の式（文字列や変数）
  戻り値: True（有効な日付として認識できる）/ False（認識できない）
  例:
    IsDate("2024/04/01")  → True
    IsDate("aaaa/bb/cc")  → False
    IsDate("1234567890")  → False
    IsDate("2024/13/01")  → False（13月は存在しない）
```

**レビュー指摘例**:
```

【分類】仕様不一致
【重要度】高
【内容】設計仕様書 FR-004 では「IsDate関数を使用して日付形式チェックを行う」
        と定義されているが、実装では Len(txtHireDate.Text) <> 10 による
        文字列長チェックのみで判定している。
        "aaaa/bb/cc" や "1234567890" などの10文字の不正な値が
        日付として通過してしまい、後続の CDate() でエラーが発生するか、
        不正なデータが登録されてしまう。
【修正案】Len() による文字列長チェックを IsDate(txtHireDate.Text) による
        日付妥当性チェックに変更してください。
```

## バグの見つけ方のコツ

1. **仕様書の太字や強調部分は特に注意して確認する**
   - 今回のバグはすべて仕様書で太字になっていた箇所に対応する

2. **ユーティリティ関数の中身も必ず確認する**
   - バグ1はフォーム（frmMain.frm）ではなくモジュール（modEmployee.bas）にある
   - 呼び出し元だけでなく、呼び出し先の実装も確認することが重要

3. **既に用意されている関数が使われているかを確認する**
   - バグ2では `FindEmployeeIndex` 関数が用意されているのに重複チェックで使われていない
   - バグ3では VB6の標準関数 `IsDate` が使われていない

4. **VB6特有の関数は引数の順序に注意する**
   - `DateDiff(interval, date1, date2)` は date2 - date1 の差分を返す
   - date1（開始日）とdate2（終了日）を逆にするとマイナスの値になる

5. **「動いている」ことと「正しい」ことは違う**
   - バグ3の `Len() = 10` チェックは動作はするが、仕様で求められるチェックではない
   - エラーハンドラが不正値を拾っているように見えても、それはバリデーションではない
