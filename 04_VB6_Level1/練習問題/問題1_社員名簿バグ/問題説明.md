# 練習問題1: 社員名簿管理アプリのバグを見つけよう

## 状況設定
10年以上前に作られたVB6アプリ「社員名簿管理」の保守案件に参加することになりました。
前任者から引き継いだソースコードのレビューを行い、仕様書との差異を見つけることがあなたのタスクです。

VB6のソースファイルはテキスト形式で保存されています。VB6 IDEがなくてもテキストエディタ（メモ帳、VSCode等）で読むことができます。コードレビューの訓練として取り組んでください。

## VB6ファイルの読み方

| ファイル | 内容 |
|---------|------|
| `EmployeeManager.vbp` | プロジェクトファイル。どのファイルが含まれるか確認できる |
| `frmMain.frm` | メインフォーム。前半がコントロール定義（画面レイアウト）、後半がコード |
| `modEmployee.bas` | 標準モジュール。型定義とユーティリティ関数 |

## あなたのタスク
1. `設計仕様書/社員名簿管理仕様書.md` を読む
2. `modEmployee.bas` を読む（型定義とユーティリティ関数を把握）
3. `frmMain.frm` のコード部分を読む（イベントプロシージャを確認）
4. 仕様書と実装の差異（バグ）を見つける
5. レビュー指摘を書く

## ヒント
- このコードには **3つのバグ** が仕込まれています
- すべて「設計仕様書の機能仕様」と実装が異なるバグです
- `modEmployee.bas` のユーティリティ関数も注意して確認してください
- VB6の日付関数 `DateDiff` と `IsDate` の使い方に着目してください

## VB6 IDEがある場合の確認方法
1. `EmployeeManager.vbp` をダブルクリックしてVB6 IDEで開く
2. F5キーでデバッグ実行
3. 設計仕様書の通りに操作してみる
4. 仕様と異なる動作をする箇所がバグです

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: プロシージャ名
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
```
