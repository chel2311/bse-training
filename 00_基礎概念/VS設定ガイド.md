# Visual Studio 設定ガイド（BSE向け）

> コードレビュー訓練を始める前に、開発環境を整える。このガイドに沿ってセットアップすれば準備完了。

---

## 1. Visual Studio 2022 のインストール

### ダウンロード

1. [Visual Studio 2022 Community](https://visualstudio.microsoft.com/ja/vs/community/) にアクセスする
2. 「無料ダウンロード」をクリックする
3. Community版は個人学習・小規模チームでの利用は無料

### ワークロードの選択

インストーラーが起動したら、以下のワークロードにチェックを入れる。

| ワークロード | 必須/任意 | 用途 |
|------------|----------|------|
| **.NETデスクトップ開発** | **必須** | C#, VB.NETのWinFormsアプリ開発・実行に必要 |
| C++によるデスクトップ開発 | 任意 | C++の問題（05_CPP_Level1）に取り組む場合 |

**補足**: ワークロードは後からでも追加できる。まずは「.NETデスクトップ開発」だけ入れて始めてよい。

### インストール後の初回設定

1. Visual Studioを初めて起動すると、開発設定の選択画面が表示される
2. 「全般」を選択する（特定の言語に偏らない標準レイアウト）
3. 配色テーマは好みで選ぶ（「濃色」が目に優しい、「淡色」が紙の資料に近い）

## 2. プロジェクトの開き方

### 方法1: .slnファイルをダブルクリック

最も簡単な方法。各問題フォルダにある `.sln`（ソリューションファイル）をエクスプローラーでダブルクリックする。

```
例: 01_CSharp_Level1/練習問題/問題1_顧客追加バグ/CustomerManager/CustomerManager.sln
```

### 方法2: Visual Studioから開く

1. Visual Studioを起動する
2. 「ファイル」→「開く」→「プロジェクト/ソリューション」を選択する
3. 対象の `.sln` ファイルを選択して「開く」をクリックする

### ソリューションエクスプローラーの見方

プロジェクトを開くと、右側に「ソリューションエクスプローラー」が表示される。

```
ソリューション 'CustomerManager'
  └─ CustomerManager（プロジェクト）
       ├─ Dependencies（依存ライブラリ）
       ├─ Program.cs（エントリーポイント）
       ├─ FormMain.cs（メインフォーム）
       ├─ FormMain.Designer.cs（フォームのデザイナーコード）
       └─ Customer.cs（顧客クラス）
```

- **Dependencies**: NuGetパッケージや参照ライブラリ。通常はレビュー不要
- **Program.cs**: アプリの起動処理。通常はレビュー不要
- **FormMain.cs**: メインの処理コード。**レビューの主な対象**
- **FormMain.Designer.cs**: 画面のレイアウト定義。イベントハンドラの接続を確認する際に見る
- **Customer.cs**: データクラス。プロパティの型・名前を確認する

## 3. 基本的なUI構成

Visual Studio 2022のウィンドウ構成を把握しておく。

```
+------------------------------------------------------------------+
|  メニューバー（ファイル/編集/表示/ビルド/デバッグ/...）           |
+------------------------------------------------------------------+
|           |                              |                        |
| ツール    |                              | ソリューション         |
| ボックス  |     コードエディタ（中央）    | エクスプローラー       |
| （左側）  |                              | （右側）               |
|           |                              |                        |
| WinForms  |     ここでコードを読む       | ファイル一覧           |
| デザイナー|                              |                        |
| 用        |                              +------------------------+
|           |                              | プロパティ             |
|           |                              | ウィンドウ（右下）     |
+-----------+------------------------------+------------------------+
|  出力 / エラー一覧 / イミディエイト ウィンドウ（下部）           |
+------------------------------------------------------------------+
```

| 領域 | 役割 | よく使う場面 |
|------|------|-------------|
| メニューバー | 全機能へのアクセス | ビルド、デバッグ開始 |
| ソリューションエクスプローラー（右側） | プロジェクト内のファイル一覧 | レビュー対象ファイルを開く |
| コードエディタ（中央） | ソースコードの表示・編集 | コードの読解 |
| プロパティウィンドウ（右下） | 選択した要素のプロパティ | コントロールの設定確認 |
| 出力/エラー一覧ウィンドウ（下部） | ビルド結果、エラーメッセージ | ビルドエラーの確認 |
| ツールボックス（左側） | WinFormsのコントロール一覧 | デザイナー使用時 |

**ウィンドウが見つからない場合**: メニューの「表示」から各ウィンドウを表示できる。

## 4. ビルドと実行

### 基本操作

| 操作 | ショートカット | 説明 |
|------|--------------|------|
| ビルド | Ctrl+Shift+B | コードをコンパイルする。エラーがあれば「エラー一覧」に表示される |
| デバッグ実行 | F5 | ビルドして実行。ブレークポイントで止まる |
| デバッグなしで実行 | Ctrl+F5 | ビルドして実行。ブレークポイントは無視される |

### ビルドエラーの見方

ビルドに失敗すると、ウィンドウ下部の「エラー一覧」にエラーが表示される。

```
エラー一覧:
  CS1002  ; が必要です    FormMain.cs  行45
  CS0103  名前 'custmer' は現在のコンテキストに存在しません    FormMain.cs  行52
```

- **CSxxxx**: C#コンパイラのエラーコード。番号でWeb検索すると原因がわかる
- **ファイル名と行番号**: ダブルクリックすると該当行にジャンプする
- **エラー内容**: 日本語で表示される（日本語パックが入っている場合）

**補足**: この教材のバグは「ビルドは通るが動作がおかしい」タイプが多い。ビルドエラーが出る場合は環境設定の問題の可能性がある。

## 5. WinFormsデザイナー

### .csファイルと.Designer.csファイルの関係

WinFormsアプリでは、1つのフォームが2つのファイルに分かれている。

| ファイル | 内容 | レビュー対象 |
|---------|------|-------------|
| FormMain.cs | 処理のコード（ボタンクリック時の処理等） | 主なレビュー対象 |
| FormMain.Designer.cs | 画面レイアウトの定義コード（自動生成） | イベントハンドラの接続確認 |

### デザイナー画面の開き方

1. ソリューションエクスプローラーでフォームファイル（例: FormMain.cs）をダブルクリックする
2. デザイナー画面が開き、フォームのレイアウトが表示される
3. コード画面に切り替えるには、フォームを右クリック →「コードの表示」、またはF7キー

### コントロールのプロパティ確認

1. デザイナー画面でコントロール（ボタン、テキストボックス等）をクリックする
2. 右下の「プロパティ」ウィンドウに、そのコントロールの設定が表示される
3. Name（名前）、Text（表示テキスト）、Enabled（有効/無効）等を確認できる

### イベントハンドラの確認方法

「ボタンにクリックイベントが接続されているか」を確認する方法。

**方法1: プロパティウィンドウから確認**
1. デザイナー画面でボタンをクリックする
2. プロパティウィンドウの上部にあるイナズマアイコン（イベント）をクリックする
3. 「Click」の欄にメソッド名（例: btnAdd_Click）が設定されていれば接続済み

**方法2: Designer.csで確認**
1. FormMain.Designer.cs を開く
2. `InitializeComponent()` メソッド内を確認する
3. 以下のような行があれば接続済み

```csharp
this.btnAdd.Click += new System.EventHandler(this.btnAdd_Click);
```

この行がない場合、ボタンを押しても何も起きない（問題1で出題されるバグパターン）。

## 7. Git統合

Visual Studio 2022にはGit機能が組み込まれている。

### Git Changesウィンドウ

メニューの「表示」→「Git Changes」で開く。変更されたファイルの一覧と差分を確認できる。

### 基本操作

| 操作 | 手順 |
|------|------|
| clone | 「Git」→「リポジトリのクローン」→ URLを入力 |
| commit | Git Changesでファイルをステージ → メッセージ入力 →「コミット」 |
| push | Git Changesの「Push」ボタン |
| pull | Git Changesの「Pull」ボタン |
| ブランチ切り替え | ウィンドウ右下のブランチ名をクリック → 切り替え先を選択 |

### 差分の確認

Git Changesで変更ファイルをダブルクリックすると、変更前後の差分が表示される。コードレビューでは、この差分表示を使って「何が変更されたか」を確認する。

## 8. SQL Server 関連

C#のDB接続問題（問題4）やSQL Server問題（07_SQLServer_Level1）で必要。

### SQL Server Express のインストール

1. [SQL Server Express](https://www.microsoft.com/ja-jp/sql-server/sql-server-downloads) にアクセスする
2. 「Express」の「今すぐダウンロード」をクリックする
3. 「基本」インストールを選択する（最も簡単）
4. インストール完了後、接続文字列が表示されるのでメモしておく

### SQL Server Management Studio (SSMS) のインストール

1. [SSMS ダウンロードページ](https://learn.microsoft.com/ja-jp/sql/ssms/download-sql-server-management-studio-ssms) からダウンロードする
2. インストーラーを実行し、デフォルト設定でインストールする

### 接続方法

1. SSMSを起動する
2. 接続ダイアログで以下を入力する

| 項目 | 値 |
|------|-----|
| サーバーの種類 | データベースエンジン |
| サーバー名 | `localhost\SQLEXPRESS` または `(localdb)\MSSQLLocalDB` |
| 認証 | Windows認証 |

3. 「接続」をクリックする

**補足**: SQL Server Expressをインストールしていない場合、`(localdb)\MSSQLLocalDB`（LocalDB）がVisual Studioと一緒にインストールされていることがある。まずはこちらで試すのが手軽。

## 9. VBA環境

VBA問題（03_VBA_Level1）で必要。Visual Studioは不要で、Excelだけあれば取り組める。

### Excelのマクロ有効化設定

1. Excelを開く
2. 「ファイル」→「オプション」→「トラスト センター」→「トラスト センターの設定」を開く
3. 「マクロの設定」で「**すべてのマクロを有効にする**」を選択する（学習中のみ。終わったら戻す）
4. 「OK」で閉じる

**セキュリティ上の注意**: 「すべてのマクロを有効にする」は学習環境でのみ使用する。業務では「警告を表示してすべてのマクロを無効にする」に戻すこと。

### VBE（Visual Basic Editor）の開き方

1. Excelでマクロ有効ブック（.xlsm）を開く
2. **Alt+F11** を押す → VBEが開く
3. 左側の「プロジェクトエクスプローラー」で標準モジュールを展開する

### .basファイルのインポート方法

教材に.bas（標準モジュール）ファイルが含まれている場合。

1. VBEを開く（Alt+F11）
2. 「ファイル」→「ファイルのインポート」を選択する
3. 対象の .bas ファイルを選択する
4. プロジェクトエクスプローラーの「標準モジュール」に追加される

## 10. トラブルシューティング

### 「.NETランタイムが見つかりません」

**原因**: .NET 8 SDKがインストールされていない。

**対処**:
1. [.NET 8 SDK ダウンロードページ](https://dotnet.microsoft.com/ja-jp/download/dotnet/8.0) にアクセスする
2. 「SDK」の列から、使用しているOSに合ったインストーラーをダウンロードする
3. インストール後、Visual Studioを再起動する

### 「プロジェクトの読み込みに失敗しました」

**原因**: 必要なワークロードがインストールされていない。

**対処**:
1. Visual Studio Installerを起動する（Windowsスタートメニューから検索）
2. 「変更」をクリックする
3. 「.NETデスクトップ開発」にチェックを入れる
4. 「変更」ボタンでインストールする
5. Visual Studioを再起動する

### ビルドエラー「CSxxxx」の調べ方

1. エラー一覧に表示されるエラーコード（例: CS0103）をコピーする
2. Webブラウザで「CS0103 C#」と検索する
3. Microsoft公式ドキュメントに原因と対処法が記載されている

### ソリューションエクスプローラーが表示されない

**対処**: メニューの「表示」→「ソリューション エクスプローラー」をクリックする（Ctrl+Alt+L）

### デバッグ実行してもウィンドウが表示されない

**対処**:
1. ソリューションエクスプローラーでプロジェクトを右クリックする
2. 「プロパティ」を開く
3. 「出力の種類」が「Windowsアプリケーション」になっているか確認する
