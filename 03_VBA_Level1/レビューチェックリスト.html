VBA Excel マクロ コードレビューチェックリスト（BSE向け）

使い方

コードレビュー時に以下の項目を1つずつ確認していく。
各項目を確認したら「OK」「NG」「N/A（該当なし）」を記入する。

1. 仕様との整合性

No 確認項目 結果
1-1 入力シートの列構成（データ型・順序）は仕様書通りか
1-2 出力シートの列構成・書式は仕様書通りか
1-3 計算ロジック（合計、平均、金額計算等）は仕様書通りか
1-4 メッセージの文言（MsgBox、InputBox）は仕様書通りか
1-5 データの抽出条件（日付範囲、フィルタ条件等）は仕様書通りか
1-6 合計行・ヘッダ行の位置と内容は仕様書通りか

2. Range操作・セル参照

No 確認項目 結果
2-1 最終行の取得方法は適切か（Cells(Rows.Count, col).End(xlUp).Row）
2-2 ループの開始行はヘッダを飛ばしているか（2行目開始）
2-3 セル参照にハードコーディングされた固定行番号がないか
2-4 Range指定とCells指定が混在して混乱していないか
2-5 クリア範囲はUsedRangeまたはCells.Clearで全体をカバーしているか

3. シート操作

No 確認項目 結果
3-1 シート参照に Set キーワードを使っているか
3-2 シート名の指定は正しいか（全角/半角、スペース）
3-3 ActiveSheetに依存せず、明示的にシート変数を使っているか
3-4 対象シートの存在チェックが必要な場合、実装されているか
3-5 複数シート間のデータ転記でシート参照を取り違えていないか

4. 変数・型宣言

No 確認項目 結果
4-1 Option Explicit がモジュールの先頭にあるか
4-2 金額に適切な型（Currency）を使っているか
4-3 整数に Long を使っているか（Integer は16bitで上限32,767）
4-4 意図しない Variant 型の変数がないか
4-5 変数の初期化は適切か

5. エラーハンドリング

No 確認項目 結果
5-1 On Error GoTo ErrorHandler が実装されているか
5-2 Exit Sub がErrorHandlerの手前にあるか
5-3 On Error Resume Next が広範囲に使われていないか
5-4 エラー時にユーザーにメッセージが表示されるか
5-5 エラー時に Application.ScreenUpdating = True に戻しているか

6. 入力チェック

No 確認項目 結果
6-1 InputBoxのキャンセル処理（空文字チェック）があるか
6-2 セルの値が空やNull、非数値の場合の考慮があるか
6-3 日付型への変換前に IsDate チェックがあるか
6-4 数値型への変換前に IsNumeric チェックがあるか

7. パフォーマンス

No 確認項目 結果
7-1 処理中に Application.ScreenUpdating = False で画面更新を停止しているか
7-2 大量データ時にセル1つずつではなく配列で一括処理しているか（大規模の場合）
7-3 不要な Select / Activate を使っていないか
7-4 処理完了後に Application.ScreenUpdating = True に戻しているか

8. 書式設定

No 確認項目 結果
8-1 ヘッダ行のフォント設定（太字等）は仕様通りか
8-2 合計行のフォント設定（太字等）は仕様通りか
8-3 日付の表示形式は仕様通りか
8-4 金額の表示形式（桁区切り等）は仕様通りか

VBA特有の頻出バグパターン

パターン 説明 確認方法
固定範囲クリア Range("A1:E50") のように固定範囲でクリアし、データが残る UsedRangeを使っているか確認
Integer溢れ 32,767を超える値でオーバーフロー Long型を使っているか確認
ActiveSheet依存 シートを明示せずにCells/Rangeを使い、意図しないシートを操作 すべてのCells/Rangeにシート変数が付いているか確認
On Error Resume Next乱用 エラーを無視して処理を続行し、異常データが蓄積 Resume Nextの影響範囲を確認
Exit Sub忘れ ErrorHandlerの手前にExit Subがなく、正常時もエラー処理に入る Sub構造全体を確認
Set忘れ オブジェクト代入時にSetを忘れ、実行時エラー オブジェクト変数への代入にSetがあるか確認
