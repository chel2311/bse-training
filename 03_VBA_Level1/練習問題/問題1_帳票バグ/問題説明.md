# 練習問題1: 月次売上帳票マクロのバグを見つけよう

## 状況設定
オフショア開発チームから「月次売上帳票マクロ」の実装が納品されました。
BSEであるあなたは、設計仕様書と照らし合わせてコードレビューを行う必要があります。

## あなたのタスク
1. `設計仕様書/月次売上帳票仕様書.md` を読む
2. `SalesReport.bas` のVBAコードを読む
3. 仕様書と実装の差異（バグ）を見つける
4. レビュー指摘を書く

## ヒント
- このコードには **3つのバグ** が仕込まれています
- すべて「設計仕様書の機能仕様」と実装が異なるバグです
- 仕様書のFR-001～FR-008と実装を1つずつ照合してください

## Excelでの確認方法
1. 新規Excelブック（.xlsm）を作成する
2. シート名を「売上データ」に変更し、`SalesData_Sample.csv` のデータを貼り付ける
3. もう1つシートを追加し、シート名を「月次帳票」にする
4. `Alt + F11` でVBEを開く
5. 「挿入」→「標準モジュール」で新しいモジュールを追加
6. `SalesReport.bas` の内容をモジュールに貼り付ける（またはファイル→インポート）
7. `Alt + F8` でマクロ一覧から「GenerateMonthlyReport」を選択して実行
8. InputBoxに「2024/01」と入力して結果を確認
9. 設計仕様書の通りに動作しているか検証する

## テストのポイント
- `SalesData_Sample.csv` のE列（売上金額）には意図的に計算ミスが含まれています
- 帳票の合計行の書式を目視で確認してください
- 2回連続でマクロを実行して、前回のデータが残っていないか確認してください

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
```
