### バグ1: 在庫金額の計算が加算になっている

**該当仕様**: FR-004 在庫金額計算
> **在庫金額 = 在庫数 x 単価**

**該当コード**: `Product.vb` 12行目付近

```vb
' 現在の実装（バグ）
Public ReadOnly Property StockValue As Integer
    Get
        Return Stock + UnitPrice
    End Get
End Property
' → 在庫数50、単価800の場合、正しくは40,000だが850になってしまう
```

**修正コード**:
```vb
Public ReadOnly Property StockValue As Integer
    Get
        Return Stock * UnitPrice
    End Get
End Property
```

```
        実装では加算（Stock + UnitPrice）で計算されている。
        例: 在庫数50、単価800の場合、正しくは40,000円だが850円と表示される。
        在庫金額合計も全て誤った値になるため、業務に重大な影響を与える。
【修正案】Product.vb の StockValue プロパティの計算式を
        「Return Stock + UnitPrice」から「Return Stock * UnitPrice」に変更してください。
```

### バグ2: 在庫警告の表示処理がない

**該当仕様**: FR-008 在庫警告表示
> **在庫数が5以下の商品は、DataGridViewの該当行を黄色（Color.Yellow）で表示する**

**該当コード**: `FormMain.vb` RefreshGrid メソッド内（170行目付近）

```vb
' 現在の実装（バグ）
Private Sub RefreshGrid(displayList As List(Of Product))
    dgvInventory.DataSource = Nothing
    dgvInventory.DataSource = displayList.Select(
        Function(p) New With {
            p.Id,
            p.ProductName,
            p.Category,
            p.Stock,
            p.UnitPrice,
            p.StockValue
        }).ToList()

    ' ← ここに在庫警告の行色変更処理がない

    UpdateStatus(displayList)
End Sub
```

**修正コード**:
```vb
Private Sub RefreshGrid(displayList As List(Of Product))
    dgvInventory.DataSource = Nothing
    dgvInventory.DataSource = displayList.Select(
        Function(p) New With {
            p.Id,
            p.ProductName,
            p.Category,
            p.Stock,
            p.UnitPrice,
            p.StockValue
        }).ToList()

    ' 列ヘッダーの表示名を設定
    If dgvInventory.Columns.Count > 0 Then
        dgvInventory.Columns("Id").HeaderText = "商品ID"
        dgvInventory.Columns("ProductName").HeaderText = "商品名"
        dgvInventory.Columns("Category").HeaderText = "カテゴリ"
        dgvInventory.Columns("Stock").HeaderText = "在庫数"
        dgvInventory.Columns("UnitPrice").HeaderText = "単価"
        dgvInventory.Columns("StockValue").HeaderText = "在庫金額"
    End If

    ' 在庫警告: 在庫数5以下の行を黄色で表示
    For Each row As DataGridViewRow In dgvInventory.Rows
        Dim stockValue As Integer = CInt(row.Cells("Stock").Value)
        If stockValue <= 5 Then
            row.DefaultCellStyle.BackColor = Color.Yellow
        Else
            row.DefaultCellStyle.BackColor = Color.White
        End If
    Next

    UpdateStatus(displayList)
End Sub
```

**レビュー指摘例**:
```
【指摘箇所】FormMain.vb: RefreshGrid メソッド内
【分類】仕様不一致
【重要度】中
【内容】設計仕様書 FR-008 では「在庫数が5以下の商品は行を黄色で表示する」と
        定義されているが、実装にはこの処理が存在しない。
        在庫が少ない商品を視覚的に把握できず、発注漏れのリスクがある。
【修正案】RefreshGrid メソッド内で DataGridView の各行をループし、
        Stock列の値が5以下の行の DefaultCellStyle.BackColor を
        Color.Yellow に設定する処理を追加してください。
```

### バグ3: 単価バリデーションの閾値が誤っている

**該当仕様**: FR-007 入力バリデーション
> 単価: **必須。1以上の整数。整数でない場合または1未満の場合はエラー「単価は1以上の整数で入力してください」**

**該当コード**: `FormMain.vb` btnAdd_Click メソッド内（55行目付近）

```vb
' 現在の実装（バグ）
Dim price As Integer
If Not Integer.TryParse(txtPrice.Text, price) OrElse price < 0 Then
    MessageBox.Show("単価は0以上の整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning)
    Return
End If
' → price < 0 のチェックなので、単価0円が登録できてしまう
' → エラーメッセージも「0以上」と誤っている
```

**修正コード**:
```vb
Dim price As Integer
If Not Integer.TryParse(txtPrice.Text, price) OrElse price < 1 Then
    MessageBox.Show("単価は1以上の整数で入力してください", "入力エラー", MessageBoxButtons.OK, MessageBoxIcon.Warning)
    Return
End If
```

```
【修正案】btnAdd_Click および btnEdit_Click の単価チェック条件を
        「price < 0」から「price < 1」に変更し、
        エラーメッセージを「単価は1以上の整数で入力してください」に修正してください。
```

## バグの見つけ方のコツ

1. **計算ロジックは必ず手計算で検証する**
   - バグ1: 在庫数50 x 単価800 = 40,000円 になるべきところ、実行すると850円になる
   - 演算子の誤り（+と*の間違い）は見た目では気付きにくい

2. **仕様書の太字・強調部分は特に注意して確認する**
   - 今回のバグはすべて仕様書で太字になっていた箇所
   - 太字 = 重要仕様 = テスト必須項目

3. **境界値でテストする**
   - バグ3: 単価に「0」を入力して追加ボタンを押してみる
   - 「以上」「以下」「未満」「超」の区別は頻出のバグポイント

4. **「ない」ことに気付く力を養う**
   - バグ2: 在庫警告の処理自体が存在しない
   - 仕様書のチェックリストを作って1つずつ潰していくのが確実
