# 練習問題1: 在庫管理アプリのバグを見つけよう

## 状況設定
オフショア開発チームから「在庫管理アプリ」の初期実装が納品されました。
BSEであるあなたは、設計仕様書と照らし合わせてコードレビューを行う必要があります。

## あなたのタスク
1. `設計仕様書/在庫管理アプリ設計仕様書.md` を読む
2. `InventoryManager/` フォルダのソースコードを読む
3. 仕様書と実装の差異（バグ）を見つける
4. レビュー指摘を書く

## ヒント
- このコードには **3つのバグ** が仕込まれています
- すべて「設計仕様書の機能仕様」と実装が異なるバグです
- FormMain.vb と Product.vb を重点的に確認してください

## Visual Studioでの確認方法
1. `InventoryManager.sln` をダブルクリックしてVisual Studioで開く
2. F5キーでデバッグ実行
3. 設計仕様書の通りに操作してみる
4. 仕様と異なる動作をする箇所がバグです

## レビュー指摘の書き方
見つけたバグは以下のフォーマットで書いてみてください:

```
【指摘箇所】ファイル名: 行番号付近
【分類】バグ / 仕様不一致
【重要度】高 / 中 / 低
【内容】設計仕様書では○○と定義されているが、実装では△△となっている。
【修正案】△△を□□に変更してください。
```
